\chapter{ภาคผนวก ฉ: แพ็กเกจการประเมิน MCP เส้นทาง IRR และข้อความอ้างอิงสำหรับ LLM}
\label{app:mcp_irr_pack}

ภาคผนวกนี้รวบรวมองค์ประกอบสำคัญของแพ็กเกจประเมิน LLM/MCP ที่ใช้ในงานวิจัย ได้แก่ ชุดสถานการณ์สำหรับการตัดสินการเลือกเครื่องมือ MCP แนวทางให้คะแนนระหว่างผู้ประเมิน ข้อความอ้างอิงสำหรับการวัดความใกล้เคียงเชิงความหมาย และกรอบการอ่านบันทึกเวลาแฝงในเส้นทาง AI assistant

\section*{องค์ประกอบของแพ็กเกจประเมิน}

แพ็กเกจประเมินนี้เชื่อมโยงกับไฟล์ผลการประเมินอัตโนมัติและ template สำหรับผู้ประเมินมนุษย์ โดยมีองค์ประกอบหลัก 4 ส่วนคือ

\begin{itemize}
    \item ชุดสถานการณ์ที่กำหนดบริบทคำถามและบริบทผู้ใช้ก่อนให้โมเดลหรือผู้ประเมินตัดสินใจ
    \item นิยาม \texttt{gold\_tool} หรือเครื่องมืออ้างอิงที่คาดว่าเหมาะสมกับสถานการณ์นั้น
    \item template สำหรับผู้ประเมินมนุษย์เพื่อใช้คำนวณ inter-rater reliability
    \item บันทึกเวลาแฝงและผลข้อความตอบเพื่อใช้สรุป proxy agreement, cosine similarity และ latency percentile
\end{itemize}

\section*{ชุดสถานการณ์สำหรับ MCP routing}

สถานการณ์ในรอบประเมินที่ใช้อยู่ในเล่มครอบคลุมอย่างน้อยกรณีตัวอย่างเช่น \texttt{dev\_light\_on}, \texttt{dev\_tv\_off}, \texttt{chat\_greeting}, \texttt{schedule\_add}, \texttt{rag\_style\_health}, และ \texttt{multi\_device} แต่ละสถานการณ์ระบุคำถามของผู้ใช้และเครื่องมือที่ควรถูกเลือกเป็นอย่างแรก เช่น \texttt{e\_device\_control}, \texttt{chat\_message}, \texttt{schedule\_modifier}, หรือ \texttt{rag\_query}

การกำหนดสถานการณ์ในระดับนี้ทำให้ผู้วิจัยสามารถประเมินได้ทั้งความถูกต้องของการเลือกเครื่องมือและความคงเส้นคงวาของการตอบข้อความ โดยไม่ปะปนกับงานปลายเปิดที่ไม่มีคำตอบอ้างอิงชัดเจน

\section*{แนวทาง IRR และการใช้ template ผู้ประเมิน}

เอกสาร metadata ของผลการประเมินระบุชัดว่า Cohen's kappa แบบสมบูรณ์ต้องอาศัยผู้ประเมินมนุษย์อย่างน้อยสองคนและใช้ template ที่แยกไว้ใน \texttt{mcp\_llm/eval/irr\_rater\_template.csv} ดังนั้นผลในบทที่~\ref{chapter4} จึงแยก ``proxy routing agreement'' ที่คำนวณจากบันทึกอัตโนมัติออกจากค่า \(\kappa\) ระหว่างผู้ประเมิน เพื่อหลีกเลี่ยงการสรุปเกินกว่าหลักฐานที่มีจริง

ในเชิงปฏิบัติ ผู้ประเมินควรอ่านข้อความสถานการณ์เดียวกัน เลือกเครื่องมือที่เหมาะสมที่สุดจากชุดเครื่องมือที่อนุญาต และใช้กฎ adjudication เดียวกันเมื่อเกิดความไม่ตรงกัน จากนั้นจึงค่อยคำนวณ \(\kappa\) หรือสถิติความสอดคล้องอื่นตามกรอบการประเมินที่กำหนดไว้

\section*{ข้อความอ้างอิงสำหรับการวัดความใกล้เคียงเชิงความหมาย}

สำหรับเมตริกความใกล้เคียงเชิงข้อความ ระบบใช้ embedding model \texttt{all-MiniLM-L6-v2} และคำนวณ cosine similarity ระหว่างข้อความตอบของโมเดลกับข้อความอ้างอิงที่จัดเตรียมไว้ โดยข้อความอ้างอิงควรรักษาองค์ประกอบสำคัญของสารสนเทศ เช่น สิ่งที่ระบบพบ สถานะล่าสุด แหล่งข้อมูลที่อ้างถึง และข้อจำกัดด้านความปลอดภัยของคำตอบ

ผลการวัดในบทที่~\ref{chapter4} จึงควรตีความว่าเป็นการตรวจสอบความใกล้เคียงเชิงใจความของข้อความ ไม่ใช่การตรวจสอบว่าข้อความต้องตรงกันทุกคำ และเหมาะกับงานที่ต้องการดูว่าระบบยังสื่อสารสาระสำคัญได้สม่ำเสมอเพียงใดภายใต้หลายรอบประเมิน

\section*{กรอบการอ่านเวลาแฝง}

ข้อมูลเวลาแฝงในผลการประเมินถูกแบ่งเป็นช่วงสำคัญ เช่น จากคำถามสู่การเริ่มเรียกเครื่องมือ จากการเรียกเครื่องมือสู่ผลลัพธ์ของเครื่องมือ จากผลลัพธ์สุดท้ายของเครื่องมือไปสู่ workflow done และจากคำถามไปสู่คำตอบสุดท้าย ข้อมูลเหล่านี้ถูกสรุปเป็นค่าเฉลี่ย, p50 และ p95 เพื่อให้ผู้อ่านเห็นทั้งแนวโน้มกลางและขอบเขตกรณีช้า

การจัดโครงสร้างเวลาแฝงแบบแยกช่วงช่วยให้สามารถวิเคราะห์ได้ว่าคอขวดเกิดในชั้นใด เช่น การรอรอบโมเดล การประมวลผลเครื่องมือ หรือการรวมผลตอบกลับเข้าสู่ข้อความสุดท้าย ซึ่งมีประโยชน์โดยตรงต่อข้อเสนอแนะในบทที่~\ref{chapter5}

\section*{ข้อควรระวังด้านข้อมูลและการทำซ้ำผล}

เมื่อมีการจัดเก็บบทสนทนาและบันทึกประเมิน ควรหลีกเลี่ยงการเก็บข้อมูลระบุตัวบุคคลโดยไม่จำเป็น และควรปิดบังชื่อบุคคล ชื่อห้อง หรือข้อมูลบริบทที่ละเอียดอ่อนหากต้องเผยแพร่แพ็กเกจประเมินต่อสาธารณะ การทำซ้ำผลควรรักษาโครงสร้างของสถานการณ์และกติกาการประเมินให้สม่ำเสมอ แม้จะเปลี่ยนค่าจริงหรือบริบทจำลองของสภาพแวดล้อม
