\chapter{ภาคผนวก ก: คลังซอร์สโค้ดและแนวทางเรียนรู้เพิ่มเติม}
\label{app:github_repo}

ภาคผนวกนี้จัดทำขึ้นเพื่อระบุแหล่งอ้างอิงเชิงวิศวกรรมของระบบต้นแบบที่ใช้ประกอบการพัฒนา
WheelSense และเพื่อเป็นแนวทางสำหรับผู้อ่านที่ประสงค์จะศึกษารายละเอียดเชิงเทคนิคเพิ่มเติมจาก
ซอร์สโค้ด เอกสารกำกับระบบ และประวัติการพัฒนา ทั้งนี้ ภาคผนวกมิได้ทำหน้าที่แทนบทที่ 3 หรือบทที่ 4
ซึ่งอธิบายวิธีการพัฒนาและผลการทดสอบไว้โดยตรง หากแต่ทำหน้าที่ชี้แหล่งข้อมูลต้นทางที่สามารถ
ตรวจสอบย้อนกลับได้ในระดับ implementation

\begin{center}
\url{https://github.com/NnopponS/WheelSense_Project}
\end{center}

สำหรับการอ้างอิงซอร์สโค้ดในช่วงจัดทำวิทยานิพนธ์ ผู้จัดทำใช้งานแท็ก \texttt{v1.0-thesis}
เป็นจุดอ้างอิงหลัก เพื่อให้ไฟล์โค้ด เอกสารประกอบ และองค์ประกอบที่ใช้ในการทดสอบสอดคล้องกับ
สาระที่นำเสนอในเล่ม อย่างไรก็ตาม คลังซอร์สโค้ดดังกล่าวยังคงมีพัฒนาการต่อเนื่องในลำดับถัดไป
ผู้อ่านจึงควรตรวจสอบเลขแท็กหรือ commit ที่ใช้อ้างอิงทุกครั้งก่อนนำข้อมูลไปใช้ต่อยอด

\section*{ก.1 วัตถุประสงค์ของการเผยแพร่คลังซอร์สโค้ด}

การเปิดเผยคลังซอร์สโค้ดมีวัตถุประสงค์หลัก 3 ประการ ดังนี้
\begin{enumerate}
    \item เพื่อให้สามารถตรวจสอบโครงสร้างระบบต้นแบบได้ในระดับที่ละเอียดกว่าที่นำเสนอในตัวเล่ม เช่น
    โครงสร้างบริการ การจัดการสิทธิ์การเข้าถึง รูปแบบการสื่อสารของอุปกรณ์ และการจัดเก็บข้อมูล
    เชิงเวลา
    \item เพื่อสนับสนุนการทำซ้ำ การทวนสอบ และการต่อยอดงานวิจัย โดยเปิดโอกาสให้ผู้อ่านศึกษาทั้ง
    ส่วนเฟิร์มแวร์ ส่วนบริการส่วนกลาง ส่วนติดต่อผู้ใช้ และเอกสารกำกับการติดตั้งในแหล่งเดียวกัน
    \item เพื่อใช้เป็นฐานความรู้สำหรับผู้สนใจด้านระบบ IoT เพื่อการดูแลสุขภาวะ การประยุกต์ใช้
    MQTT และ REST ร่วมกัน การออกแบบแดชบอร์ดเชิงปฏิบัติการ และการประสานงานกับผู้ช่วย
    ปัญญาประดิษฐ์ภายใต้กรอบความปลอดภัย
\end{enumerate}

\section*{ก.2 แนวทางการศึกษาคลังซอร์สโค้ด}

เพื่อให้การศึกษาคลังซอร์สโค้ดเป็นระบบ ผู้อ่านควรเริ่มจากเอกสารภาพรวมก่อน แล้วจึงพิจารณา
รายละเอียดในแต่ละส่วนย่อยตามลำดับของชั้นสถาปัตยกรรม กล่าวคือ เริ่มจากเอกสารภาพรวมของระบบ
จากนั้นศึกษาส่วนอุปกรณ์ภาคสนาม บริการส่วนกลาง ส่วนติดต่อผู้ใช้ และสุดท้ายจึงพิจารณาเอกสารกำกับ
การทดสอบหรือเอกสารเชิงปฏิบัติการอื่น ๆ วิธีการเช่นนี้ช่วยให้เห็นความสัมพันธ์ระหว่างองค์ประกอบต่าง ๆ
ของระบบได้ชัดเจนกว่าการอ่านเฉพาะไฟล์โค้ดรายไฟล์

ในเชิงปฏิบัติ ผู้อ่านควรให้ความสำคัญกับเอกสาร 4 กลุ่ม ได้แก่
\begin{enumerate}
    \item เอกสารภาพรวมสถาปัตยกรรมและ runtime truth ซึ่งอธิบายชั้นของระบบ บทบาทของบริการ
    หลัก และเส้นทางการไหลของข้อมูล
    \item เอกสารกำกับบริการส่วนกลาง ซึ่งอธิบาย API, MQTT ingestion, การจัดการสิทธิ์ และชั้น
    การประสานงาน MCP/AI
    \item เอกสารเฟิร์มแวร์และแอปพลิเคชันผู้ใช้ ซึ่งช่วยให้เข้าใจข้อจำกัดของอุปกรณ์จริง รูปแบบ payload
    และข้อพิจารณาด้านประสบการณ์ผู้ใช้
    \item เอกสารด้านการทดสอบและการปฏิบัติการ ซึ่งใช้ทำความเข้าใจวิธีตั้งระบบ การจำลองข้อมูล
    และการตรวจสอบสถานะของบริการในสภาพแวดล้อมทดลอง
\end{enumerate}

\section*{ก.3 แนวทางเรียนรู้เพิ่มเติม}

สำหรับผู้อ่านที่ต้องการศึกษาต่อยอด beyond ตัวเล่ม สามารถแบ่งแนวทางเรียนรู้เพิ่มเติมได้เป็น
4 สายหลักดังนี้
\begin{enumerate}
    \item \textbf{สายระบบฝังตัวและอุปกรณ์ภาคสนาม} ควรศึกษาวิธีการอ่านค่าจาก IMU, การสแกน
    BLE, การจัดการพลังงานของอุปกรณ์แบบสวมใส่ และการออกแบบ payload สำหรับการส่งข้อมูล
    ผ่านเครือข่ายที่มีข้อจำกัด
    \item \textbf{สายระบบบริการส่วนกลาง} ควรศึกษาการออกแบบบริการด้วย FastAPI, การจัดการ
    asynchronous I/O, การทำงานร่วมกันระหว่าง REST และ MQTT, การออกแบบฐานข้อมูลเชิงเวลา
    และการทำระบบผ่าน container
    \item \textbf{สายส่วนติดต่อผู้ใช้และ HMI} ควรศึกษาการออกแบบแดชบอร์ดเชิงปฏิบัติการ,
    การจัดลำดับข้อมูลสำคัญบนหน้าจอ, การลดภาระการรับรู้ของผู้ใช้, และการออกแบบส่วนติดต่อ
    ที่สอดคล้องกับบทบาทผู้ใช้หลายกลุ่ม
    \item \textbf{สาย AI และการประสานงานเชิงปลอดภัย} ควรศึกษาการใช้ local LLM,
    กลไกการจำกัดสิทธิ์ผ่าน MCP, การออกแบบ propose-confirm-execute, และการทำ audit trail
    สำหรับการกระทำที่ผู้ช่วยปัญญาประดิษฐ์มีส่วนเกี่ยวข้อง
\end{enumerate}

กล่าวโดยสรุป ภาคผนวกนี้มีเป้าหมายเพื่อชี้ให้เห็นว่า WheelSense มิได้เป็นเพียงผลลัพธ์เชิงเอกสาร
แต่มีฐานเชิง implementation ที่ตรวจสอบได้จริงในคลังซอร์สโค้ดสาธารณะ และยังสามารถใช้เป็น
แหล่งศึกษาต่อยอดสำหรับงานวิจัยและงานพัฒนาระบบในอนาคตได้
