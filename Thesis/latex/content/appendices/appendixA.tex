\chapter{ภาคผนวก ก: โครงสร้างคลังซอร์สโค้ดและแนวทางการติดตั้งระบบต้นแบบ}

ภาคผนวกนี้สรุปโครงสร้างหลักของคลังซอร์สโค้ด WheelSense และแนวทางติดตั้งระบบต้นแบบในระดับที่เพียงพอสำหรับการทำซ้ำผลหรือการตรวจสอบองค์ประกอบของแพลตฟอร์มร่วมกับบทที่~\ref{chapter3} และบทที่~\ref{chapter4}

\section*{โครงสร้างระดับบนของคลังซอร์สโค้ด}

โครงงานจัดเก็บองค์ประกอบหลักไว้ในคลังเดียวเพื่อให้พัฒนาระบบต้นแบบทั้งฝั่งอุปกรณ์ ฝั่งบริการ และเอกสารวิจัยไปพร้อมกันได้ โดยโครงสร้างระดับบนที่สำคัญมีดังนี้

\begin{itemize}
    \item \texttt{frontend/} สำหรับเว็บแดชบอร์ดและส่วนติดต่อผู้ใช้หลักของแพลตฟอร์ม
    \item \texttt{server/} สำหรับ FastAPI backend, ฐานข้อมูล, MQTT ingestion, AI runtime และ MCP server
    \item \texttt{firmware/} สำหรับซอร์สโค้ดอุปกรณ์ภาคสนาม เช่น อุปกรณ์เก้าอี้รถเข็นและโหนดกล้อง
    \item \texttt{mobile-app/} สำหรับแอปมือถือที่ใช้เชื่อมกับอุปกรณ์สวมใส่และเสริมช่องทางใช้งานนอกเว็บ
    \item \texttt{Thesis/} สำหรับไฟล์ \LaTeX{}, รูปประกอบ, สคริปต์สร้างกราฟ และเอกสารประกอบการเขียนเล่ม
    \item \texttt{docs/} และ \texttt{server/docs/} สำหรับเอกสารสถาปัตยกรรม สภาพแวดล้อม และคู่มือการดำเนินงาน
\end{itemize}

การแยกโครงสร้างในลักษณะนี้ทำให้สามารถดูเส้นทางข้อมูลตั้งแต่ firmware ไปยัง MQTT, backend, dashboard และ AI assistant ได้ชัดเจน และเอื้อต่อการอธิบายสถาปัตยกรรมในบทที่~\ref{chapter3} โดยไม่ต้องกระจายรายละเอียดไปหลายคลังแยกกัน

\section*{ข้อกำหนดเบื้องต้นสำหรับการติดตั้ง}

จากเอกสารปฏิบัติการของฝั่งเซิร์ฟเวอร์ ระบบต้นแบบต้องการองค์ประกอบหลักอย่างน้อยดังนี้

\begin{itemize}
    \item Python 3.12 หรือรุ่นที่เข้ากันได้สำหรับ backend และสคริปต์ช่วยงาน
    \item Docker Desktop หรือ Docker Engine สำหรับรันฐานข้อมูล Mosquitto และบริการประกอบอื่น
    \item PostgreSQL ผ่าน Docker Compose สำหรับฐานข้อมูลหลักของแพลตฟอร์ม
    \item Node.js เมื่อต้องการรันหรือพัฒนาส่วน \texttt{frontend/}
    \item เครื่องมือ \LaTeX{} เช่น \texttt{xelatex} และ \texttt{biber} เมื่อต้องการคอมไพล์เล่มวิทยานิพนธ์
\end{itemize}

\section*{ลำดับการติดตั้งระบบต้นแบบโดยสังเขป}

ลำดับการติดตั้งที่ใช้ได้กับสภาพแวดล้อมพัฒนาและสาธิตต้นแบบสามารถสรุปได้ดังนี้

\begin{enumerate}
    \item เตรียมไฟล์ตั้งค่าพื้นฐานของเซิร์ฟเวอร์จาก \texttt{server/.env.example}
    \item เริ่มบริการฐานข้อมูลและ MQTT broker ด้วย Docker Compose
    \item อัปเกรด schema ฐานข้อมูลด้วย Alembic ให้ตรงกับ migration ล่าสุด
    \item เริ่ม backend service และตรวจสอบว่า API, MQTT listener และ MCP endpoint ทำงานครบ
    \item เริ่มเว็บแดชบอร์ดและตรวจสอบการเชื่อมกับ API
    \item ลงทะเบียนอุปกรณ์ภาคสนามหรือเปิดโหมด auto-register ตามสภาพแวดล้อมทดลอง
    \item เชื่อมแอปมือถือและอุปกรณ์สวมใส่เมื่อต้องการทดสอบสายงาน Polar/mobile telemetry
\end{enumerate}

แนวทางดังกล่าวช่วยให้ผู้ทำซ้ำผลสามารถเริ่มจากสภาพแวดล้อมแบบจำลองก่อน แล้วจึงค่อยเชื่อมอุปกรณ์จริงและข้อมูลจริงในลำดับถัดไป ซึ่งลดความเสี่ยงต่อการสับสนระหว่างปัญหาการตั้งค่าระบบกับปัญหาเชิงตรรกะของแอปพลิเคชัน

\section*{โหมดการใช้งานหลักของ backend}

เอกสารฝั่งเซิร์ฟเวอร์ระบุการใช้งานสองโหมดหลักคือโหมด production database และโหมด mock/simulator ซึ่งใช้ Compose entry คนละชุดแต่คงโครงบริการหลักร่วมกัน การแยกโหมดดังกล่าวมีประโยชน์ต่อการวิจัยเพราะทำให้สามารถทดสอบตรรกะระบบและการไหลของข้อมูลในฐานข้อมูลตัวอย่างได้ก่อน โดยไม่ปะปนกับฐานข้อมูลที่ใช้ในสภาพแวดล้อมใช้งานจริง

ในโหมด simulator ระบบสามารถใช้ฐานข้อมูล mock ที่เตรียมไว้พร้อมตัวอย่างผู้ใช้ ผู้ป่วย อุปกรณ์ และตัวจำลอง MQTT ส่วนโหมด production database ใช้ฐานข้อมูลสะอาดสำหรับการติดตั้งจริงหรือการสาธิตที่ต้องการสภาพข้อมูลควบคุมเอง ความแตกต่างนี้สอดคล้องกับลักษณะการประเมินในบทที่~\ref{chapter4} ซึ่งบางส่วนเน้นการยืนยันเส้นทางข้อมูลและบางส่วนเน้นพฤติกรรมเชิงระบบภายใต้ภาระงานต้นแบบ

\section*{จุดสังเกตด้านการทำซ้ำผล}

เพื่อให้การทำซ้ำผลมีความใกล้เคียงกับงานวิจัยในเล่ม ควรเก็บข้อมูลเวอร์ชันของ migration, โหมดฐานข้อมูลที่ใช้งาน, รายการอุปกรณ์ที่ลงทะเบียน, และไฟล์ตั้งค่าที่เกี่ยวข้องกับ AI/MQTT ไว้ทุกครั้ง การบันทึกบริบทเหล่านี้มีความสำคัญมากเป็นพิเศษเมื่อระบบประกอบด้วยหลายโมดูลที่สามารถเปลี่ยนสถานะการทำงานได้จากทั้งโค้ดและค่าตั้งสภาพแวดล้อม

ภาคผนวกนี้จึงทำหน้าที่เป็นสะพานระหว่างโครงสร้าง repository จริงกับคำอธิบายเชิงสถาปัตยกรรมในเล่ม เพื่อให้ผู้อ่านที่สนใจด้านวิศวกรรมสามารถมองเห็นขอบเขตการติดตั้งและการดูแลระบบต้นแบบได้ชัดเจนขึ้น

\section*{คลังซอร์สโค้ดออนไลน์}

\begin{center}
\url{https://github.com/NnopponS/WheelSense_Project}
\end{center}

ผู้สนใจสามารถ clone คลังซอร์สโค้ดด้วยคำสั่ง \texttt{git clone} และดำเนินการตามแนวทางการติดตั้งที่ระบุไว้ข้างต้น tag ของเวอร์ชันที่ใช้ในงานวิจัยนี้คือ \texttt{v1.0-thesis} ซึ่งตรึงสถานะของโค้ดไว้ ณ วันที่ส่งปริญญานิพนธ์
