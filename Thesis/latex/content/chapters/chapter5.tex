\section{สรุปผลโครงงาน}

โครงงาน WheelSense มุ่งพัฒนาระบบต้นแบบสภาพแวดล้อมอัจฉริยะสำหรับผู้ใช้เก้าอี้รถเข็น โดยบูรณาการอุปกรณ์ตรวจจับการเคลื่อนที่ ระบบระบุตำแหน่งภายในอาคาร และแพลตฟอร์มซอฟต์แวร์สำหรับแสดงผลและวิเคราะห์ข้อมูล ผลการดำเนินงานแสดงให้เห็นว่าระบบสามารถรับข้อมูลจากอุปกรณ์ปลายทาง ประมวลผลร่วมกับข้อมูลตำแหน่ง และแสดงผลบนแดชบอร์ดได้จริงในระดับต้นแบบ

\subsection{ผลสำเร็จตามวัตถุประสงค์หลัก}

\begin{enumerate}
    \item พัฒนาอุปกรณ์ตรวจจับการเคลื่อนที่บน M5StickC Plus2 ร่วมกับ IMU และเชื่อมข้อมูลสรีรศาสตร์จาก Polar Verity Sense ได้
    \item พัฒนาการติดตามตำแหน่งภายในอาคารด้วยแนวทาง BLE-based positioning โดยให้ M5StickC Plus2 ส่งข้อมูล RSSI ขึ้นเซิร์ฟเวอร์และใช้ fingerprinting ร่วมกับ KNN บนเซิร์ฟเวอร์ได้ในระดับการใช้งานต้นแบบ
    \item พัฒนาโครงสร้างรับส่งข้อมูลและบริการฝั่งเซิร์ฟเวอร์บน Raspberry Pi 5 (8GB RAM) ได้ครบสายงานข้อมูลหลัก
    \item พัฒนา application สำหรับติดตามสถานะและข้อมูลแบบเรียลไทม์ด้วย Next.js, React และ Tailwind CSS ได้
    \item บูรณาการ Home Assistant และ local LLM (Gemma 4B) ผ่าน MCP สำหรับการช่วยตีความข้อมูลได้ในระดับต้นแบบ
\end{enumerate}

\section{ปัญหาและอุปสรรค}

\subsection{ข้อจำกัดเชิงเทคนิค}

ประเด็นหลักที่พบระหว่างการทดสอบระบบจริงคือคุณภาพสัญญาณ RSSI ที่เปลี่ยนตามสภาพแวดล้อมและการปรับพารามิเตอร์ของโมเดล KNN ให้เหมาะกับแต่ละพื้นที่ ซึ่งส่งผลต่อความเสถียรของผล localization ในบางช่วงเวลา

\subsection{ข้อจำกัดเชิงข้อมูลและการใช้งานระบบ}

พบความท้าทายด้านการจัดการ timestamp ระหว่างองค์ประกอบต่าง ๆ ทำให้เกิดความคลาดเคลื่อนของลำดับเหตุการณ์ในบางช่วง และกระทบการวิเคราะห์ย้อนหลัง อีกทั้งขั้นตอนการติดตั้ง/ตั้งค่าระบบยังซับซ้อนและใช้เวลาสูง จึงต้องปรับปรุงกระบวนการ deployment ให้เป็นมาตรฐานมากขึ้น

\section{ข้อเสนอแนะในการพัฒนาต่อ}

\subsection{การปรับสถาปัตยกรรมระบบ}

แนวทางหลักคือการเพิ่มความเสถียรของ localization pipeline โดยปรับวิธีเก็บ fingerprint dataset ให้ครอบคลุมสภาพแวดล้อมจริงมากขึ้น และพัฒนาวิธีปรับพารามิเตอร์ KNN ตามบริบทการใช้งาน เพื่อให้ผลตำแหน่งคงที่และเชื่อถือได้ยิ่งขึ้น

\subsection{การพัฒนา AI และระบบอัตโนมัติ}

ควรพัฒนาโมดูล machine learning และ local LLM (Gemma 4B) ที่ใช้งานผ่าน MCP ให้แม่นยำและครอบคลุมกรณีใช้งานมากขึ้น โดยเฉพาะการตีความเหตุการณ์ผิดปกติจากข้อมูล accelerometer แบบ rule-based และการแปลงผลวิเคราะห์เป็นคำแนะนำที่นำไปใช้ได้จริง พร้อมเชื่อมต่อ Home Assistant เพื่อสร้างการตอบสนองอัตโนมัติที่เหมาะสมกับบริบทการใช้งานจริง

\subsection{การประเมินผลระยะยาว}

ควรขยายการทดลองกับผู้ใช้งานจริงที่หลากหลายและเพิ่มระยะเวลาทดสอบต่อเนื่อง เพื่อประเมินเสถียรภาพ ความน่าเชื่อถือ และประโยชน์เชิงคุณภาพชีวิตในสถานการณ์จริง รวมถึงนำข้อเสนอแนะจากผู้เชี่ยวชาญมาปรับปรุงเกณฑ์การตีความพฤติกรรมและลดการแจ้งเตือนผิดพลาดในระบบ
