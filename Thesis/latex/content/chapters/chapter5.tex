\chapter{สรุปผลการวิจัย และข้อเสนอแนะ}
\label{chapter5}

\section{อภิปรายผลงานวิจัย}
\label{sec:ch5_discussion}

ในงานวิจัยนี้ เราได้พัฒนาระบบต้นแบบสภาพแวดล้อมอัจฉริยะสำหรับผู้ใช้เก้าอี้รถเข็น โดยบูรณาการข้อมูลจากอุปกรณ์ภาคสนาม ระบบระบุตำแหน่งภายในอาคาร บริการส่วนกลาง และผู้ช่วยปัญญาประดิษฐ์ (AI) ภายใต้กรอบความปลอดภัย ระบบออกแบบมาเพื่อตอบสนองความต้องการในการติดตามและเฝ้าระวังผู้สูงอายุในสถานดูแลอย่างต่อเนื่อง โดยใช้ข้อมูลจากเซ็นเซอร์บนรถเข็น อุปกรณ์สวมใส่ และระบบระบุตำแหน่งเพื่อสร้างภาพสถานการณ์แบบเรียลไทม์ และใช้ AI เพื่อช่วยผู้ดูแลในการตัดสินใจ

ด้านระบบเซ็นเซอร์บนรถเข็น ผลการทดสอบฮาร์ดแวร์ M5StickC Plus2 แสดงว่าอุปกรณ์สามารถสุ่มข้อมูล IMU ได้ที่อัตรา 19.6 Hz ซึ่งใกล้เคียงเป้าหมาย 20 Hz ความครบถ้วนของ telemetry อยู่ที่ 98.4\% ซึ่งแสดงว่าระบบส่งข้อมูลได้อย่างต่อเนื่องผ่าน MQTT broker เวลาแฝงจากอุปกรณ์ถึง broker เฉลี่ย 41 ms (P50) ซึ่งอยู่ในระดับที่ยอมรับได้สำหรับการติดตามแบบเรียลไทม์ การตรวจจับสถานะการเคลื่อนไหวด้วย F1-Score 0.89--0.95 แสดงว่าอัลกอริทึมสามารถแยกแยะสถานะนิ่ง เคลื่อนที่ช้า และเคลื่อนที่เร็วได้ดี การใช้ dead-band filter และ low-pass filter ช่วยลด false positive จากสัญญาณรบกวน ผลลัพธ์เหล่านี้สอดคล้องกับงานวิจัยที่เกี่ยวข้องกับการตรวจจับกิจกรรมด้วย IMU ซึ่งมักใช้ค่า sampling rate ในช่วง 10--50 Hz แต่งานวิจัยนี้แสดงให้เห็นว่าการใช้ dead-band filter สามารถลดการแจ้งเตือนที่ไม่จำเป็นได้อย่างมีประสิทธิภาพ

สำหรับระบบระบุตำแหน่ง ผลการทดสอบ BLE fingerprinting ด้วย KNN ($k=5$) ให้ค่า accuracy 0.85 ที่ระดับโซนหรือห้อง ซึ่งเพียงพอสำหรับงานเฝ้าติดตามของผู้ดูแล แต่ยังมีข้อจำกัดเมื่อมีสิ่งกีดขวางหรือคนจำนวนมากในพื้นที่ (RSSI drift) เมทริกซ์ความสับสนแสดงว่าความผิดพลาดส่วนใหญ่เกิดระหว่างโซนที่อยู่ติดกัน ซึ่งมีลักษณะ RSSI คล้ายคลึงกัน ผลลัพธ์นี้สอดคล้องกับงานวิจัยก่อนหน้าที่แสดงให้เห็นว่า BLE fingerprinting มีความแม่นยำในระดับห้อง แต่มีข้อจำกัดเมื่อสภาพแวดล้อมเปลี่ยนแปลง งานวิจัยนี้แสดงให้เห็นว่าการใช้ KNN กับ $k=5$ เป็นทางเลือกที่เหมาะสมสำหรับระดับความซับซ้อนของสถานดูแล แต่อาจต้องการการปรับปรุงเพิ่มเติมด้วยเทคนิคเช่น Kalman filter เพื่อเพิ่มความเสถียรของการ tracking

อุปกรณ์สวมใส่ที่ใช้ในการทดสอบคือ Polar Verity Sense ซึ่งแสดงว่าค่า HR มีความคลาดเคลื่อนอยู่ในช่วงที่ยอมรับได้ตามมาตรฐาน AAMI EC13 การเชื่อมต่อ BLE GATT มีอัตราสำเร็จ 98.6\% พร้อม auto-reconnect การประมวลผลข้อมูลสัญญาณชีพสามารถตรวจจับความผิดปกติของ HR ได้ด้วยความแม่นยำ 82\% ผลลัพธ์เหล่านี้แสดงให้เห็นว่าอุปกรณ์สวมใส่ที่มีอยู่ในตลาดสามารถนำมาใช้ในบริบทสถานดูแลได้ โดยไม่ต้องพัฒนาฮาร์ดแวร์เฉพาะทาง ซึ่งช่วยลดต้นทุนและเพิ่มความง่ายในการนำไปใช้จริง อย่างไรก็ตาม การตรวจจับความผิดปกติของ HR ยังมีความแม่นยำไม่สูงพอสำหรับการใช้ในการวินิจฉัยทางการแพทย์ แต่เพียงพอสำหรับการเฝ้าระวังเบื้องต้น

การทดสอบ deployment บน Raspberry Pi 5 สำหรับเซิร์ฟเวอร์ส่วนกลาง แสดงว่า MQTT broker เริ่มเร็วที่สุด (5.8 วินาที) ในขณะที่ MCP runtime ใช้เวลานานที่สุด (31.5 วินาที) เนื่องจากต้องโหลดโมเดล Gemma 4 เข้าสู่หน่วยความจำ ผลลัพธ์นี้แสดงให้เห็นว่าการใช้ LLM ในเครื่อง (on-prem) มีข้อจำกัดด้านเวลาเริ่มต้น แต่เมื่อทำงานแล้วสามารถให้บริการได้อย่างต่อเนื่อง การทดสอบ Home Assistant Integration ผ่าน MCP Protocol มีอัตราสำเร็จ 98.2\% สำหรับการควบคุมหลอดไฟ และ 94.6\% สำหรับการสั่งถ่ายภาพกล้อง เวลาแฝงคำสั่ง device control อยู่ที่ 0.42 วินาที (P50) การทดสอบความปลอดภัยแสดงว่าระบบสามารถบังคับใช้ RBAC และ Scope Narrowing ได้ 100\% AI เข้าถึงเฉพาะ tool ที่อนุญาตตามบทบาท ผลลัพธ์เหล่านี้แสดงให้เห็นว่า MCP Protocol สามารถใช้เชื่อมต่อ AI กับระบบ IoT ที่มีอยู่แล้วได้อย่างปลอดภัย ซึ่งแตกต่างจากงานวิจัยก่อนหน้าที่มักใช้ API โดยตรงโดยไม่มีกรอบควบคุมความปลอดภัย

ระบบผู้ช่วยอัจฉริยะที่ใช้ EaseAI Pipeline ผลการทดสอบแสดงว่าชั้น L4 (LLM Synthesis) ใช้เวลามากที่สุด (P95 = 118.5 วินาที) คิดเป็น 99.9\% ของเวลาทั้งหมด เนื่องจากการประมวลผลบน Raspberry Pi 5 มีข้อจำกัดด้านฮาร์ดแวร์ Tool selection accuracy อยู่ที่ 0.86 และ Inter-rater agreement ($\kappa$) อยู่ที่ 0.81 ซึ่งแสดงว่า AI เลือกเครื่องมือได้ถูกต้องในระดับสูงและมีความเห็นตรงกันระหว่างผู้ประเมิน Action confirmation compliance อยู่ที่ 1.00 แสดงว่าระบบปฏิบัติตามขั้นตอน propose-confirm-execute ได้อย่างสมบูรณ์ ผลลัพธ์เหล่านี้แสดงให้เห็นว่าการใช้ pipeline 5 ชั้นช่วยแยกความรับผิดชอบและทำให้ระบบสามารถตรวจสอบความปลอดภัยได้ทุกขั้นตอน แต่ latency ที่สูงยังเป็นข้อจำกัดสำคัญที่ต้องแก้ไข

ส่วนติดต่อผู้ใช้งานมีคะแนน SUS เฉลี่ย 76.5 อยู่ในระดับ ``ใช้งานได้ดี'' (Good) และอยู่ในเปอร์เซ็นไทล์ 75 หมายความว่าระบบมีความใช้งานได้ดีกว่า 75\% ของระบบที่ถูกประเมินด้วยมาตรฐานเดียวกัน อัตราสำเร็จภารกิจเฉลี่ย 91.3\% แสดงว่าผู้ใช้สามารถทำงานส่วนใหญ่ได้สำเร็จ ความพึงพอใจต่อ AI อยู่ในระดับสูง (4.1/5) ผลลัพธ์เหล่านี้แสดงให้เห็นว่าการออกแบบส่วนติดต่อผู้ใช้ที่เน้นความเรียบง่ายและความชัดเจนช่วยให้ผู้ดูแลสามารถใช้งานได้โดยไม่ต้องมีพื้นฐานด้านเทคโนโลยีมากนัก ซึ่งเป็นสิ่งสำคัญสำหรับการนำไปใช้ในสถานดูแลจริง

การทดสอบแบบปลายถึงปลายด้วย 8 สถานการณ์แสดงว่าระบบสามารถทำงานเชื่อมโยงกันได้ตั้งแต่ต้นทางถึงปลายทาง สถานการณ์ที่ใช้เวลานานที่สุดคือ ``เรียกภาพบริบท'' ที่ 2.46 วินาที เนื่องจากการดาวน์โหลดภาพผ่าน MQTT ต้องใช้เวลาในการรวมชิ้นส่วนภาพ ผลลัพธ์นี้แสดงให้เห็นว่าแม้แต่ละส่วนของระบบจะทำงานได้ดีเมื่อทดสอบแยกกัน แต่การทำงานร่วมกันยังมีความท้าทาย โดยเฉพาะการส่งข้อมูลขนาดใหญ่ผ่าน MQTT ซึ่งอาจต้องการการปรับปรุงโปรโตคอลหรือการบีบอัดข้อมูล

\section{สรุปผลงานวิจัย}
\label{sec:ch5_summary}

วิทยานิพนธ์นี้พัฒนาระบบต้นแบบสภาพแวดล้อมอัจฉริยะสำหรับผู้ใช้เก้าอี้รถเข็ก บูรณาการข้อมูลจากอุปกรณ์ภาคสนาม ระบบระบุตำแหน่งภายในอาคาร บริการส่วนกลาง และผู้ช่วย AI ภายใต้กรอบความปลอดภัย ผลในบทที่~\ref{chapter4} แสดงว่าระบบสามารถทำงานได้ต่อเนื่องในระดับต้นแบบ ในส่วนของระบบเซ็นเซอร์บนรถเข็ก ความครบถ้วนของ telemetry อยู่ที่ 98.4\% เวลาแฝง 41 ms (P50) และการตรวจจับการเคลื่อนไหว F1-Score 0.89--0.95 ในส่วนของระบบระบุตำแหน่ง accuracy อยู่ที่ 0.85 ที่ระดับโซนหรือห้องด้วย KNN ($k=5$) ในส่วนของอุปกรณ์สวมใส่ HR accuracy อยู่ในระดับที่ยอมรับได้ตามมาตรฐาน AAMI EC13 และ BLE GATT connection มีอัตราสำเร็จ 98.6\% ในส่วนของเซิร์ฟเวอร์ส่วนกลาง ระบบตอบสนองได้สม่ำเสมอภายใต้ normal load และยังทำงานได้ต่อเนื่องภายใต้ stress load โดย Home Assistant integration มีอัตราสำเร็จ 98.2\% ในส่วนของระบบผู้ช่วย AI Tool selection accuracy อยู่ที่ 0.86 Inter-rater agreement $\kappa$ อยู่ที่ 0.81 และ Action confirmation compliance อยู่ที่ 1.00 ในส่วนของส่วนติดต่อผู้ใช้ คะแนน SUS อยู่ที่ 76.5 (ระดับใช้งานได้ดี) อัตราสำเร็จภารกิจ 91.3\% และความพึงพอใจ AI 4.1/5

\section{ข้อจำกัดของงานวิจัย}
\label{sec:ch5_limitations}

งานวิจัยนี้มีข้อจำกัดด้านเทคนิค คือ ความแปรผันของ RSSI ภายในอาคารส่งผลต่อ localization ทำให้ accuracy ลดลงเมื่อมีสิ่งกีดขวางหรือคนจำนวนมากในพื้นที่ ทรัพยากร local LLM จำกัดทำให้ latency สูงขึ้นเมื่อใช้งานพร้อมกัน (P95 = 118.5 วินาที) ในด้านการประเมิน ชุดข้อมูลทดสอบยังไม่ครอบคลุมความหลากหลายของสภาพแวดล้อมจริงทั้งหมด การประเมิน UX ใช้ผู้เข้าร่วม 15 คน (ผู้ดูแลจริง 5 คน ผู้จำลองบทบาท 10 คน) เพียงพอสำหรับระดับต้นแบบแต่ยังไม่มากพอสำหรับการสรุปผลเชิงประชากร ในด้านความน่าเชื่อถือของผลการวิจัย ความไม่สม่ำเสมอของเครือข่ายและสภาพสัญญาณ อาจส่งผลต่อความถูกต้องของข้อมูล การทดสอบในบริบทจำลองเป็นหลัก และตัวชี้วัดบางส่วนยังขึ้นอยู่กับการออกแบบสถานการณ์

\section{ข้อเสนอแนะในการวิจัย}
\label{sec:ch5_recommendations}

ข้อเสนอแนะเชิงวิศวกรรม: ปรับปรุง fingerprinting และเพิ่ม Kalman filter เพื่อเพิ่มความเสถียรของ localization เพิ่ม health check และการสำรองข้อมูล ปรับปรุง UI ให้ย่อย่อและเพิ่ม shortcut สำหรับงานที่ใช้บ่อย คงแนวทาง propose-confirm-execute และพิจารณาใช้ LLM ที่เล็กกว่าหรือ GPU external เพื่อลด latency

ข้อเสนอแนะเชิงงานวิจัยต่อยอด: ขยายการทดลองไปสถานดูแลจริงหลายแห่ง ศึกษาการผสานข้อมูลหลายแหล่งสำหรับ localization (เช่น BLE + IMU) พัฒนาวิธีประเมิน AI ที่ครอบคลุมความถูกต้อง ความปลอดภัย และผลกระทบต่อผู้ดูแล ศึกษาผลต่อภาระงานและความพึงพอใจในระยะยาว และพัฒนากรอบจริยธรรมสำหรับบริบทสถานดูแล

ข้อเสนอแนะเชิงการประยุกต์: เริ่มจากบทบาทที่มีประโยชน์ชัดเจนและเสี่ยงต่ำ เช่น การติดตามตำแหน่งระดับโซน การดูประวัติย้อนหลัง และการแจ้งเตือนเชิงบริบท จากนั้นขยายไปใช้ AI ช่วยสรุปหรือสนับสนุน workflow ระบบบูรณาการอุปกรณ์ภาคสนาม ระบบระบุตำแหน่ง บริการส่วนกลาง และผู้ช่วย AI ภายใต้กรอบควบคุมที่เหมาะสม มีศักยภาพยกระดับระบบสนับสนุนผู้ใช้เก้าอี้รถเข็น
