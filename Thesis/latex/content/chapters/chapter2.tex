\section{บริบทการดูแลผู้สูงอายุและความท้าทายในสถานดูแล}
\label{sec:ch2_care_context}

การเปลี่ยนผ่านสู่สังคมสูงวัย (Aging Society) ส่งผลกระทบโดยตรงต่อระบบการจัดการในสถานดูแลผู้สูงอายุ (Nursing Homes หรือ Care Facilities) ซึ่งต้องรองรับผู้รับบริการที่มีความเปราะบางและมีความต้องการที่หลากหลายมากขึ้น \cite{WHO2021DecadeAgeing} ในบริบทของสถานดูแลขนาดกลางถึงใหญ่ เช่น บ้านพักคนชราวาสนะเวศม์ ซึ่งมีพื้นที่กว้างขวางและมีอาคารพักอาศัยหลายหลัง ความท้าทายที่สำคัญคืออัตราส่วนระหว่างบุคลากรและผู้สูงอายุ โดยในขณะที่สถานดูแลระดับพรีเมียมอาจมีอัตราส่วนที่ 1:1 ถึง 1:4 แต่ในสถานดูแลของรัฐหรือสถานดูแลขนาดใหญ่ทั่วไป อัตราส่วนผู้ดูแลต่อผู้สูงอายุอาจสูงถึง 1:20 โดยเฉพาะในช่วงเวรกลางคืน \cite{Schulz2020CaregiverBurden, SituationThaiElderly2022} ซึ่งเป็นตัวเลขที่สะท้อนถึงวิกฤตการขาดแคลนแรงงานในภาคส่วนการดูแลผู้สูงอายุทั้งในระดับประเทศและระดับโลก การมีผู้ดูแลเพียง 1 คนต่อผู้รับบริการ 20 ท่าน ทำให้เกิด "ช่องว่างในการเฝ้าระวัง" (Monitoring Gap) ที่รุนแรง และความเสี่ยงต่อการเกิดอุบัติเหตุหรือภาวะฉุกเฉินที่ไม่ได้รับการตอบสนองทันท่วงที

\section{ทฤษฎีระบบระบุตำแหน่งภายในอาคาร}
\label{sec:ch2_ips}

เนื่องจากระบบระบุตำแหน่งผ่านดาวเทียม (GPS) มีข้อจำกัดในการใช้งานภายในอาคารเนื่องจากสัญญาณถูกบดบังด้วยโครงสร้างอาคาร ระบบระบุตำแหน่งภายในอาคาร (Indoor Positioning System: IPS) จึงถูกพัฒนาขึ้นเพื่อระบุพิกัดในพื้นที่ปิด เทคนิคการระบุตำแหน่งพื้นฐานที่แพร่หลายมีดังนี้:

\subsection{เทคนิคเชิงระยะทาง (Range-based Techniques)}
อาศัยการคำนวณระยะทางระหว่างตัวรับและตัวส่งสัญญาณ โดยเทคนิคที่แพร่หลายที่สุดคือ \textbf{Trilateration} ซึ่งคำนวณจุดตัดของวงกลม 3 วงที่เกิดจากจุดอ้างอิงอ้างอิง 3 จุด ดังรูปที่~\ref{fig:ch2_trilateration}

\begin{figure}[!ht]
    \centering
    \begin{tikzpicture}[scale=0.6]
        % Define coordinates for 3 anchors
        \coordinate (A1) at (0,0);
        \coordinate (A2) at (5,1);
        \coordinate (A3) at (2,4);
        
        % Define radii
        \def\rOne{3}
        \def\rTwo{2.5}
        \def\rThree{2.1}
        
        % Intersection point (approximate for illustration)
        \coordinate (P) at (2.4, 1.8);
        
        % Draw circles
        \draw[thick, blue!50] (A1) circle (\rOne);
        \draw[thick, red!50] (A2) circle (\rTwo);

        % Draw anchors
        \filldraw (A1) circle (2pt) node[below left] {$A_1(x_1, y_1)$};
        \filldraw (A2) circle (2pt) node[right] {$A_2(x_2, y_2)$};
        \filldraw (A3) circle (2pt) node[above] {$A_3(x_3, y_3)$};

        \filldraw[red] (P) circle (3pt) node[below right, font=\bfseries] {$P(x, y)$};

        \draw[dashed, gray] (A1) -- (P) node[midway, above left, black] {$d_1$};
        \draw[dashed, gray] (A2) -- (P) node[midway, above right, black] {$d_2$};
        \draw[dashed, gray] (A3) -- (P) node[midway, left, black] {$d_3$};
        
    \end{tikzpicture}
    \caption{หลักการระบุตำแหน่งแบบ Trilateration จากจุดอ้างอิง 3 จุด}
    \label{fig:ch2_trilateration}
\end{figure}

หลักการของ Trilateration คือการหาจุดตัดของวงกลม 3 วง โดยพิกัดเป้าหมาย $(x, y)$ สามารถคำนวณได้จากความสัมพันธ์ระหว่างพิกัดจุดอ้างอิง $(x_i, y_i)$ และระยะทาง $d_i$ ดังสมการ:

\begin{equation}
\text{พิกัด } P(x, y) \text{ คำนวณได้จากระบบสมการ:} \quad
\begin{cases}
(x - x_1)^2 + (y - y_1)^2 = d_1^2 \\
(x - x_2)^2 + (y - y_2)^2 = d_2^2 \\
(x - x_3)^2 + (y - y_3)^2 = d_3^2 
\end{cases}
\label{eq:ch2_trilateration_sys}
\end{equation}

เมื่อ $(x_i, y_i)$ คือพิกัดของจุดอ้างอิงที่ $i$ และ $d_i$ คือระยะทางที่วัดได้จากจุดอ้างอิงนั้นๆ การแก้สมการเพื่อหาพิกัด $(x, y)$ สามารถทำได้โดยการนำสมการที่ 1 ไปลบออกจากสมการที่ 2 และ 3 เพื่อกำจัดตัวแปรยกกำลังสอง จะได้ระบบสมการเชิงเส้นในรูปแบบเมทริกซ์ $AX = B$ ดังสมการ~(\ref{eq:ch2_trilateration_matrix}):

\begin{equation}
\begin{bmatrix}
2(x_2 - x_1) & 2(y_2 - y_1) \\
2(x_3 - x_1) & 2(y_3 - y_1)
\end{bmatrix}
\begin{bmatrix}
x \\
y
\end{bmatrix}
=
\begin{bmatrix}
(x_2^2 - x_1^2) + (y_2^2 - y_1^2) + (d_1^2 - d_2^2) \\
(x_3^2 - x_1^2) + (y_3^2 - y_1^2) + (d_1^2 - d_3^2)
\end{bmatrix}
\label{eq:ch2_trilateration_matrix}
\end{equation}

ซึ่งพิกัดเป้าหมาย $P(x, y)$ จะคำนวณได้จากการหาอินเวอร์สของเมทริกซ์ $A$ ต่อไป

สำหรับการหาระยะทาง ($d$) จากความแรงของสัญญาณวิทยุ มักใช้แบบจำลองการลดทอนของสัญญาณ (Log-distance Path Loss Model) ดังรูปที่~\ref{fig:ch2_rssi_dist} และสมการ~(\ref{eq:ch2_pathloss}):

\begin{figure}[!ht]
    \centering
    \begin{tikzpicture}[scale=1.1]
        % Axes
        \draw[->] (0,-4) -- (6,-4) node[right] {ระยะทาง $d$ (เมตร)};
        \draw[->] (0,-4) -- (0,0.5) node[above] {RSSI (dBm)};
        
        % Curve
        \draw[thick, blue, domain=0.5:5.5, smooth, variable=\x] plot ({\x}, {-1 - 1.5*log10(\x)});
        
        \node[blue, right] at (2.5,-2.2) {$\mathrm{RSSI}(d) = \mathrm{RSSI}(d_0) - 10n \log_{10}(d)$};
        
        % Reference point at 1m
        \filldraw (1,-1) circle (1.5pt);
        \draw[dashed] (1,-4) -- (1,-1) node[midway, left, font=\scriptsize] {$1$ ม.};
        \draw[dashed] (0,-1) -- (1,-1) node[left] {$\mathrm{RSSI}(d_0)$};
    \end{tikzpicture}
    \caption{ความสัมพันธ์ระหว่างค่า RSSI และระยะทางตามแบบจำลอง Path Loss}
    \label{fig:ch2_rssi_dist}
\end{figure}

\begin{equation}
\mathrm{RSSI}(d) = \mathrm{RSSI}(d_0) - 10n \log_{10}\left(\frac{d}{d_0}\right) + X_{\sigma}
\label{eq:ch2_pathloss}
\end{equation}
เมื่อ $\mathrm{RSSI}(d)$ คือกำลังสัญญาณที่ได้รับที่ระยะ $d$, $\mathrm{RSSI}(d_0)$ คือกำลังสัญญาณอ้างอิงที่ระยะ 1 เมตร, $n$ คือค่าสัมประสิทธิ์การสูญเสียสัญญาณในพื้นที่ และ $X_{\sigma}$ คือสัญญาณรบกวนที่มีการกระจายตัวแบบปกติ \cite{Kaemarungsi2012RSSIAnalysis}

\subsection{เทคนิคการสร้างแผนที่สัญญาณ (Fingerprinting)}
เป็นวิธีที่นิยมใช้ในอาคารที่มีโครงสร้างซับซ้อน แบ่งเป็น 2 ขั้นตอนหลัก ดังแสดงกระบวนการในรูปที่~\ref{fig:ch2_fingerprinting}:
\begin{enumerate}
    \item \textbf{Offline Phase:} เก็บข้อมูลความแรงสัญญาณ (RSSI) จากตัวส่งสัญญาณทุกจุดในแต่ละพิกัดพื้นที่ เพื่อสร้างเป็น "ลายนิ้วมือสัญญาณ" (Radio Map)
    \item \textbf{Online Phase:} นำสัญญาณที่วัดได้จริงเปรียบเทียบกับ Radio Map โดยใช้อัลกอริทึม เช่น K-Nearest Neighbors (KNN) เพื่อหาตำแหน่งที่ใกล้เคียงที่สุด
\end{enumerate}

สำหรับการคำนวณความใกล้เคียงระหว่างสัญญาณที่วัดได้แบบเรียลไทม์กับลายนิ้วมือสัญญาณในฐานข้อมูล มักใช้การหาระยะห่างแบบยูคลิด (Euclidean Distance) ดังสมการ \cite{Kaemarungsi2012RSSIAnalysis}:
\begin{equation}
D_j = \sqrt{\sum_{i=1}^{N} (\mathrm{RSSI}_{i} - \mathrm{RSSI}_{i,j})^2}
\label{eq:ch2_knn_distance}
\end{equation}
เมื่อ $D_j$ คือระยะห่างแบบยูคลิดระหว่างสัญญาณที่วัดได้กับลายนิ้วมือสัญญาณที่จุดอ้างอิง $j$, $N$ คือจำนวนจุดส่งสัญญาณ (Beacon) ทั้งหมดที่ครอบคลุม, $\mathrm{RSSI}_{i}$ คือค่าความแรงสัญญาณจาก Beacon ตัวที่ $i$ ที่วัดได้ในขณะนั้น และ $\mathrm{RSSI}_{i,j}$ คือค่าความแรงสัญญาณจาก Beacon ตัวที่ $i$ ณ จุดอ้างอิง $j$ ในฐานข้อมูล โดยระบบจะเลือกตำแหน่งอ้างอิงที่มีค่า $D_j$ ต่ำที่สุด หรือเฉลี่ยจาก $K$ จุดที่ต่ำที่สุด เพื่อใช้เป็นพิกัดเป้าหมาย

        % Online Phase
        \node[draw, rectangle, rounded corners, fill=green!10, text width=3cm, align=center, right=of offline] (online) {\textbf{Online Phase}\\เปรียบเทียบสัญญาณ\\(Pattern Matching)};

    \end{tikzpicture}

\end{figure}

\section{เทคโนโลยีเซ็นเซอร์สำหรับการตรวจวัดกิจกรรม}
\label{sec:ch2_sensors}

เทคโนโลยีเซ็นเซอร์แบบสวมใส่เป็นองค์ประกอบสำคัญในระบบติดตามและดูแลสุขภาพ แบ่งออกเป็นเซ็นเซอร์ทางสรีรวิทยา (Physiological Sensors) และเซ็นเซอร์ทางกายภาพ (Inertial Sensors)

\subsection{Photoplethysmography (PPG)}

\textbf{(1) อัตราการเต้นของหัวใจ (Heart Rate):}
\begin{equation}
\mathrm{HR} = \frac{60}{\overline{RR}}
\label{eq:ch2_hr}
\end{equation}
เมื่อ $\overline{RR}$ คือค่าเฉลี่ยของช่วง RR-interval (วินาที) และ $\mathrm{HR}$ มีหน่วยเป็น bpm (ครั้งต่อนาที)

\textbf{(2) ความแปรปรวนของอัตราการเต้นของหัวใจ (HRV) ด้วย SDNN:}
\begin{equation}
\mathrm{SDNN} = \sqrt{\frac{1}{N-1}\sum_{i=1}^{N}(RR_i - \overline{RR})^2}
\label{eq:ch2_sdnn}
\end{equation}
เมื่อ $N$ คือจำนวนช่วง RR ทั้งหมด และ $RR_i$ คือช่วง RR ที่ $i$ ค่า SDNN ที่ต่ำลงบ่งบอกถึงระบบประสาทอัตโนมัติที่ตอบสนองน้อยลง

\textbf{(3) RMSSD สำหรับประเมินกิจกรรมประสาทพาราซิมพาเทติก:}
\begin{equation}
\mathrm{RMSSD} = \sqrt{\frac{1}{N-1}\sum_{i=1}^{N-1}(RR_{i+1} - RR_i)^2}
\label{eq:ch2_rmssd}
\end{equation}
ค่า RMSSD นิยมใช้ตรวจจับความเครียดทางกายภาพ (Physical Stress) เพราะสะท้อนการควบคุมอัตราการเต้นของหัวใจในช่วงสั้น

ข้อมูลจาก IMU สามารถนำไปประมวลผลได้ดังนี้

\textbf{(1) ขนาดของความเร่งสุทธิ (Resultant Acceleration):}
\begin{equation}
|\vec{a}| = \sqrt{a_x^2 + a_y^2 + a_z^2}
\label{eq:ch2_acc_mag}
\end{equation}
ค่านี้ใช้เป็นฟีเจอร์หลักสำหรับการตรวจจับการล้ม เนื่องจากการล้มจะทำให้เกิดการเปลี่ยนแปลงของ $|\vec{a}|$ อย่างรวดเร็ว

\textbf{(2) การลด Noise จากแรงโน้มถ่วงด้วย Low-pass Filter (EMA):}
\begin{equation}
\hat{\omega}_{z,k} = \alpha \cdot \omega_{z,k} + (1 - \alpha) \cdot \hat{\omega}_{z,k-1}
\label{eq:ch2_ema_gyro}
\end{equation}
เมื่อ $\omega_{z,k}$ คืออัตราการหมุนดิบจาก Gyroscope ณ เวลา $k$, $\hat{\omega}_{z,k}$ คืออัตราการหมุนที่ผ่านการกรองแล้ว และ $\alpha$ คือค่าคงที่ของ Exponential Moving Average (EMA) โดยทั่วไปอาจกำหนด $\alpha = 0.3$ เพื่อลด noise ของ MPU6886 ในแนวแกน Z ซึ่งมีผลจากแรงโน้มถ่วงเมื่อทิศทางของเซ็นเซอร์เปลี่ยนไปตามการหมุนของล้อ นอกจากนี้ยังมีการกำหนด Deadband ที่ $|\hat{\omega}_z| < 5$ dps เพื่อตัด drift ขณะหยุดนิ่ง

\textbf{(3) การคำนวณระยะทางจาก Arc Length ของล้อ:}
\begin{equation}
\Delta s_k = |\hat{\omega}_{z,k} \cdot \Delta t| \cdot r
\label{eq:ch2_arc_length}
\end{equation}
เมื่อ $\Delta s_k$ คือระยะทางที่ล้อเคลื่อนที่ในช่วงเวลา $\Delta t$ (วินาที), $r$ คือรัศมีล้อรถเข็น (เช่น $r = 0.30$ เมตร สำหรับรถเข็นทั่วไป) และระยะทางสะสมทั้งหมดคือ $s = \sum_{k} \Delta s_k$ สมการนี้ใช้หลักการ Arc Length ($s = r\theta$) โดยที่มุมการหมุน $\Delta\theta_k = \hat{\omega}_{z,k} \cdot \Delta t$

\textbf{(4) การคำนวณความเร็วด้วย Sliding Window:}
\begin{equation}
v = \frac{\displaystyle\sum_{k \in W} \Delta s_k}{T_W}
\label{eq:ch2_velocity_window}
\end{equation}
เมื่อ $W$ คือช่วงหน้าต่างเวลา (Sliding Window) ขนาด $T_W = 500$ มิลลิวินาที และ $v$ คือความเร็วเฉลี่ยในหน้าต่างนั้น วิธีนี้ช่วยลดผลของ noise ที่สะสมจากการ integrate โดยตรง

\textbf{(5) การตรวจจับการล้ม (Fall Detection) ด้วย Threshold:}
\begin{equation}
\text{Fall} =
\begin{cases}
1 & \text{ถ้า } |\vec{a}| > T_{\mathrm{fall}} \text{ และ } |\vec{a}| < T_{\mathrm{impact}} \text{ ภายหลัง} \\
0 & \text{อื่นๆ}
\end{cases}
\label{eq:ch2_fall}
\end{equation}
โดย $T_{\mathrm{fall}}$ คือเกณฑ์ความเร่งระหว่างการร่วงหล่น (Free-fall threshold, เช่น $< 0.5g$) และ $T_{\mathrm{impact}}$ คือเกณฑ์ความเร่งขณะกระแทก (Impact threshold, เช่น $> 2g$) การตรวจพบทั้งสองเหตุการณ์ภายในช่วงเวลาสั้นจะถูกวินิจฉัยว่าเป็นการล้ม


\section{อัลกอริทึม K-Nearest Neighbors (KNN)}
\label{sec:ch2_knn}

อัลกอริทึม K-Nearest Neighbors (KNN) เป็นหนึ่งในเทคนิคการเรียนรู้แบบมีการสอน (Supervised Learning) ที่เรียบง่ายและมีประสิทธิภาพสูงสำหรับการจำแนกประเภท (Classification) และการประมาณค่า (Regression) \cite{Cover1967NearestNeighbor} โดยหลักการทำงานพื้นฐานคือการคำนวณความคล้ายคลึงระหว่างข้อมูลใหม่และข้อมูลในฐานข้อมูลผ่านฟังก์ชันระยะทาง (Distance Function) และพิจารณา "เพื่อนบ้าน" ที่อยู่ใกล้เคียงที่สุดจำนวน $K$ ตัวในพื้นที่ของคุณลักษณะ (Feature Space) เพื่อตัดสินใจเลือกคลาสที่มีจำนวนมากที่สุด ดังแสดงในรูปที่~\ref{fig:ch2_knn_concept}

ขั้นตอนการคำนวณของ KNN ประกอบด้วยสมการหลักดังนี้:

\textbf{(1) การคำนวณระยะทางระหว่างข้อมูล (Distance Calculation):}
มักนิยมใช้ระยะห่างแบบยูคลิด (Euclidean Distance) เพื่อวัดความห่างระหว่างจุดข้อมูลใหม่ $X$ และจุดข้อมูลในฐานข้อมูล $Y$ ในพื้นที่ $n$ มิติ:
\begin{equation}
d(X, Y) = \sqrt{\sum_{i=1}^{n} (x_i - y_i)^2}
\label{eq:ch2_knn_euclidean}
\end{equation}
เมื่อ $x_i$ และ $y_i$ คือค่าของคุณลักษณะ (Feature) ตัวที่ $i$ ของข้อมูล $X$ และ $Y$ ตามลำดับ

\textbf{(2) การจำแนกประเภทข้อมูล (Classification Rule):}
หลังจากหาระยะทางและเลือกเพื่อนบ้านที่ใกล้ที่สุด $K$ อันดับแรก ($N_K$) ระบบจะทำการตัดสินใจเลือกคลาส $C$ โดยใช้เสียงส่วนใหญ่ (Majority Voting):
\begin{equation}
\hat{y} = \text{mode}\{ y_i \mid i \in N_K \}
\label{eq:ch2_knn_voting}
\end{equation}
เมื่อ $\hat{y}$ คือคลาสที่ทำนายได้ และ $y_i$ คือคลาสของเพื่อนบ้านลำดับที่ $i$

\subsection{M5StickC Plus2}
M5StickC Plus2 เป็นชุดพัฒนา IoT ขนาดจิ๋วจากตระกูล M5Stack ที่รวมองค์ประกอบพื้นฐานที่จำเป็นสำหรับการสร้างอุปกรณ์สวมใส่หรือโหนดเซ็นเซอร์ไร้สายไว้ในตัวเดียว ข้อมูลทางเทคนิคระบุว่าอุปกรณ์ใช้ชิป ESP32-PICO-V3-02 (Dual-core 240 MHz) มีหน่วยความจำ Flash 8 MB, PSRAM 2 MB และจอภาพ TFT ขนาด 1.14 นิ้ว \cite{M5StickCPlus2Docs} จุดเด่นสำคัญคือการติดตั้งเซ็นเซอร์วัดการเคลื่อนไหวแบบ 6 แกน (6-axis IMU) รุ่น MPU6886 มาให้ในบอร์ด พร้อมปุ่มกด ลำโพงขนาดเล็ก และแบตเตอรี่ Li-Po ในตัว

ด้วยลักษณะที่กะทัดรัดและรองรับการสื่อสารทั้ง Wi-Fi และ Bluetooth อุปกรณ์นี้จึงนิยมใช้ในงานต้นแบบระบบติดตามตำแหน่ง การตรวจวัดกิจกรรมทางกายภาพ และการควบคุมอุปกรณ์อัจฉริยะจากระยะไกล นอกจากนี้ยังมีพอร์ตขยาย (Grove port) ที่ช่วยให้สามารถเชื่อมต่อกับเซ็นเซอร์ภายนอกเพิ่มเติมได้ง่าย ทำให้เป็นเครื่องมือที่มีความยืดหยุ่นสูงสำหรับการพัฒนาแอปพลิเคชัน IoT ที่ต้องการการตอบสนองแบบเรียลไทม์

\subsection{Lilygo T-SimCam}
Lilygo T-SimCam เป็นบอร์ดไมโครคอนโทรลเลอร์ที่ออกแบบมาเฉพาะสำหรับการประมวลผลภาพที่ขอบเครือข่าย (Visual Edge Computing) โดยใช้ชิปหลักเป็น ESP32-S3R8 ซึ่งมีความโดดเด่นในการรองรับชุดคำสั่งสำหรับการคำนวณแบบเวกเตอร์เพื่อเร่งความเร็วงานด้าน AI อุปกรณ์ประกอบด้วยหน่วยความจำ PSRAM ขนาด 8 MB และ Flash 16 MB พร้อมกล้อง OV2640 ความละเอียด 2 ล้านพิกเซล \cite{LilygoTSimCamDocs}

คุณลักษณะเด่นของอุปกรณ์คือการผสานความสามารถในการบันทึกภาพและการสื่อสารไร้สายความเร็วสูง (Wi-Fi และ Bluetooth 5) ทำให้สามารถทำหน้าที่เป็นโหนดเฝ้าระวังอัจฉริยะที่สามารถคัดกรองเหตุการณ์จากภาพ (Image-based triggering) ได้ในตัวโดยไม่ต้องส่งข้อมูลวิดีโอทั้งหมดไปยังเซิร์ฟเวอร์ นอกจากนี้ยังมีช่องเสียบการ์ดหน่วยความจำและการรองรับไมโครโฟนดิจิทัล ทำให้เป็นฮาร์ดแวร์ที่เหมาะสมสำหรับการพัฒนาระบบรักษาความปลอดภัย หรือระบบตรวจจับวัตถุเบื้องต้นที่มีข้อจำกัดด้านพลังงานและพื้นที่

\subsection{Polar Verity Sense}
Polar Verity Sense เป็นเซ็นเซอร์วัดอัตราการเต้นของหัวใจแบบออปติคอล (Optical Heart Rate Sensor) ที่ใช้เทคโนโลยี PPG (Photoplethysmography) ที่มีความแม่นยำสูง อุปกรณ์ถูกออกแบบมาให้เป็นแขนรัดหรือสวมใส่ในตำแหน่งที่ใกล้กับเส้นเลือดฝอยบนผิวหนัง รองรับการสื่อสารผ่าน Bluetooth Low Energy (BLE) และ ANT+ ได้พร้อมกัน และมีความสามารถในการทำงานต่อเนื่องสูงสุดถึง 30 ชั่วโมง \cite{Polar_VeritySense_Spec, PolarVeritySenseWhitepaper}

จุดเด่นของอุปกรณ์รุ่นนี้คือความน่าเชื่อถือของข้อมูลชีวสัญญาณและการออกแบบให้มีความทนทาน กันน้ำ และสามารถเก็บข้อมูลได้ในตัวแม้ไม่มีการเชื่อมต่อกับโทรศัพท์สมาร์ทโฟนในขณะนั้น นอกจากนี้ยังมีเซ็นเซอร์วัดการเคลื่อนไหว (Inertial Sensors) ภายในเพื่อใช้ชดเชยสัญญาณรบกวนจากการเคลื่อนที่ (Motion Artifact) ทำให้ผลลัพธ์ของอัตราการเต้นของหัวใจมีความเสถียรแม้ในขณะที่ผู้ใช้งานมีการเคลื่อนไหวร่างกายอย่างต่อเนื่อง อุปกรณ์นี้จึงถูกนำไปใช้แพร่หลายทั้งในการวิจัยทางการกีฬาและการติดตามสภาวะสุขภาพทางไกล

\subsection{Raspberry Pi 5}
Raspberry Pi 5 เป็นคอมพิวเตอร์บอร์ดขนาดเล็ก (Single Board Computer) รุ่นล่าสุดจาก Raspberry Pi Foundation ที่ใช้ชิป ARM Cortex-A76 Quad-core ความถี่ 2.4 GHz มีหน่วยความจำ RAM 8 GB และรองรับการเชื่อมต่อผ่าน Gigabit Ethernet, Wi-Fi 5, Bluetooth 5.0 และ USB 3.0 \cite{RaspberryPi5Specs} จุดเด่นของรุ่นนี้คือประสิทธิภาพการประมวลผลที่เพิ่มขึ้นอย่างมากเมื่อเทียบกับรุ่นก่อนหน้า ทำให้เหมาะสำหรับการใช้งานเป็นเซิร์ฟเวอร์ขนาดเล็กที่ต้องการการประมวลผลข้อมูลแบบเรียลไทม์และบริการต่างๆ พร้อมกัน

ด้วยขนาดที่กะทัดรัดและการใช้พลังงานที่ต่ำ Raspberry Pi 5 จึงเป็นทางเลือกที่น่าสนใจสำหรับการทำระบบเซิร์ฟเวอร์ส่วนกลางในโครงการ IoT ที่ต้องการความสามารถในการประมวลผลที่สูง แต่ยังคงความยืดหยุ่นในการติดตั้งและบำรุงรักษา อุปกรณ์นี้สามารถรันบริการต่างๆ เช่น MQTT Broker, Database Server, API Server และ AI Runtime ได้อย่างมีประสิทธิภาพในบอร์ดเดียว

\subsection{แนวคิดการออกแบบ UX/UI สำหรับระบบติดตามสภาวะ}
แนวคิด UX/UI สำหรับระบบที่ต้องการการเฝ้าระวังหรือติดตามสภาวะมีจุดมุ่งหมายหลักเพื่อให้ผู้ใช้สามารถรับรู้สถานะสำคัญและตอบสนองต่อเหตุการณ์ฉุกเฉินได้อย่างรวดเร็ว แนวคิด \textbf{User Experience (UX)} ในกลุ่มนี้มักเน้นการลดภาระทางความคิด (Cognitive Load) ของผู้ใช้ \cite{Sweller1988CognitiveLoad} โดยการจัดลำดับข้อมูลจากสิ่งที่เร่งด่วนที่สุดไว้ในตำแหน่งที่สังเกตง่าย \cite{Few2013Dashboard,Ware2012InformationVisualization} ขณะที่ \textbf{User Interface (UI)} ต้องสื่อความหมายได้ทันทีผ่านสัญลักษณ์หรือรหัสสี \cite{Gates2018AlertFatigue} และรองรับการแสดงผลบนอุปกรณ์ที่หลากหลาย (Responsive Design)

การพัฒนาแอปพลิเคชันในรูปแบบเว็บ (Web Application) เป็นทางเลือกที่มีความยืดหยุ่นสูง \cite{Nielsen1994Heuristics} เนื่องจากผู้ใช้สามารถเข้าถึงระบบผ่านเบราว์เซอร์ได้ทันทีโดยไม่ต้องติดตั้งโปรแกรมเฉพาะเครื่อง ทำให้การกระจายระบบในองค์กรทำได้ง่าย นอกจากนี้ยังรองรับการทำงานข้ามแพลตฟอร์ม ทั้งบนคอมพิวเตอร์ตั้งโต๊ะ แท็บเล็ต และสมาร์ทโฟน ซึ่งช่วยเพิ่มความต่อเนื่องในการทำงานเมื่อผู้ใช้ต้องเปลี่ยนอุปกรณ์ในระหว่างปฏิบัติงาน

\subsection{FastAPI (Backend Framework)}
FastAPI เป็นกรอบงานพัฒนาเว็บ API ที่เน้นประสิทธิภาพ ความชัดเจนของชนิดข้อมูล และการทำงานบนมาตรฐานสมัยใหม่ โดยอาศัยมาตรฐาน OpenAPI และ JSON Schema สำหรับนิยาม API และสนับสนุนการสร้างเอกสารอัตโนมัติผ่าน Swagger UI และ ReDoc \cite{FastAPI2024} คุณสมบัตินี้มีประโยชน์โดยตรงต่อการพัฒนาระบบขนาดใหญ่ที่ต้องมีการเชื่อมต่อระหว่าง Mobile App, Firmware Gateway และบริการอื่น ๆ ร่วมกันอย่างสอดคล้อง

อีกประเด็นสำคัญคือ FastAPI อาศัย Python Type Hints และ Pydantic ในการตรวจสอบข้อมูล ทำให้สามารถกำหนดสัญญา (Contract) ระหว่าง Input และ Output ได้ชัดเจนตั้งแต่ระดับโค้ดจนถึงเอกสาร API \cite{FastAPI2024,Pydantic2024} สำหรับระบบ IoT ที่ต้องรับข้อมูลจากแหล่งที่หลากหลาย ความสามารถในการตรวจสอบความถูกต้องของข้อมูลตั้งแต่ขอบเขตของระบบ (System Boundary) ช่วยลดโอกาสเกิดความผิดพลาดในการประมวลผลข้อมูลที่ผิดรูปแบบได้

\subsection{React (UI Library)}
React เป็นไลบรารีจาวาสคริปต์ (JavaScript Library) แบบโอเพนซอร์สที่ใช้สำหรับการสร้างส่วนติดต่อผู้ใช้ (User Interface) โดยอาศัยสถาปัตยกรรมแบบคอมโพเนนต์ (Component-based Architecture) \cite{React2024} แนวคิดหลักของ React คือการแบ่งหน้าจอ UI ออกเป็นส่วนย่อย ๆ ที่สามารถนำกลับมาใช้ใหม่ได้ (Reusable Components) และมีการจัดการสถานะ (State Management) ที่แยกเป็นอิสระต่อกัน

คุณสมบัติที่สำคัญของ React คือการใช้ Virtual DOM ซึ่งเป็นตัวแทนของ DOM (Document Object Model) ในหน่วยความจำ เมื่อมีการเปลี่ยนแปลงสถานะข้อมูล React จะคำนวณความแตกต่างระหว่าง Virtual DOM เดิมและโครงสร้างใหม่ แล้วอัปเดตเฉพาะส่วนที่เปลี่ยนแปลงไปยัง DOM จริงบนเบราว์เซอร์ ซึ่งช่วยเพิ่มประสิทธิภาพในการแสดงผลข้อมูลที่มีการเปลี่ยนแปลงบ่อยครั้ง เช่น แดชบอร์ดติดตามสถานะแบบเรียลไทม์ ดังแสดงในรูปที่~\ref{fig:ch2_react_vdom}

\subsection{React Native และ Expo (Mobile Development Framework)}
การพัฒนาแอปพลิเคชันบนอุปกรณ์เคลื่อนที่ (Mobile Application) เป็นส่วนสำคัญของระบบที่ต้องการการแจ้งเตือนแบบเรียลไทม์และการเข้าถึงข้อมูลระหว่างเดิน โดยเฉพาะในบริบทของสถานดูแลผู้สูงอายุที่บุคลากรต้องเคลื่อนที่ตลอดเวลา \textbf{React Native} เป็นกรอบงานที่ช่วยให้สามารถพัฒนาแอปพลิเคชันสำหรับ iOS และ Android โดยใช้โค้ดจาวาสคริปต์ร่วมกัน ลดความซับซ้อนในการดูแลโค้ดแยกตามแพลตฟอร์ม \cite{ReactNativeDocs}

\subsection{PostgreSQL (Database Management System)}
PostgreSQL เป็นระบบจัดการฐานข้อมูลเชิงสัมพันธ์ (Relational Database) ที่มีจุดเด่นในการรองรับธุรกรรมแบบ ACID และการจัดเก็บข้อมูลที่มีโครงสร้างซับซ้อนร่วมกับข้อมูลกึ่งโครงสร้างในรูปแบบ JSONB \cite{PostgreSQL2024,Obe2017_PostgreSQL_in_Action} ความสามารถนี้มีความสำคัญอย่างยิ่งต่อระบบ IoT ที่ต้องจัดการทั้งข้อมูลหลักที่มีความสัมพันธ์ชัดเจน (เช่น ข้อมูลผู้ใช้และอุปกรณ์) และข้อมูลจากเซิร์ฟเวอร์ที่มีโครงสร้างยืดหยุ่นหรือเปลี่ยนแปลงได้ตามรุ่นของฮาร์ดแวร์

การใช้ฐานข้อมูลที่รองรับทั้ง Relational และ JSONB ช่วยให้นักพัฒนาสามารถรักษาความสอดคล้องของข้อมูลหลักผ่านระบบ Schema ที่เข้มงวด ขณะเดียวกันก็สามารถขยายขีดความสามารถในการเก็บ Metadata หรือข้อมูลดิบจากเซ็นเซอร์ได้โดยไม่ต้องเปลี่ยนโครงสร้างฐานข้อมูลบ่อยครั้ง นำไปสู่การออกแบบระบบที่มีความเสถียรและยืดหยุ่นต่อการขยายผลในอนาคต

\subsection{Docker (Containerization)}
Docker มีบทบาทสำคัญในการสร้างความสม่ำเสมอของสภาพแวดล้อมการทำงาน (Runtime Environment) โดยการใช้คอนเทนเนอร์เพื่อแยกแต่ละองค์ประกอบของแอปพลิเคชันออกจากกัน \cite{DockerContainerDocs2026} แนวคิดนี้ช่วยแก้ปัญหาความแตกต่างของ Dependency ระหว่างเครื่องนักพัฒนาและเครื่องเซิร์ฟเวอร์จริง ทำให้การปรับใช้ระบบ (Deployment) สามารถทำได้อย่างรวดเร็วและลดโอกาสเกิดข้อผิดพลาดจากสภาพแวดล้อมที่ไม่ตรงกัน

\subsection{Ollama และ Large Language Models (LLM)}
การนำโมเดลภาษาขนาดใหญ่ (LLM) มาใช้ในระบบติดตามพฤติกรรมมีวัตถุประสงค์เพื่อช่วยสรุปเหตุการณ์และสนับสนุนการโต้ตอบเชิงภาษาธรรมชาติ อย่างไรก็ตาม ระบบลักษณะนี้มีข้อกำหนดด้านความเป็นส่วนตัวสูง จึงมีความจำเป็นต้องใช้ Local Inference มากกว่าบริการภายนอก \textbf{Ollama} เป็นแพลตฟอร์มที่ช่วยให้การรันโมเดล LLM ภายในโครงสร้างพื้นฐานขององค์กรทำได้สะดวก พร้อมเปิด API ที่แอปพลิเคชันอื่นเรียกใช้ได้โดยตรง \cite{Ollama2024,OllamaAPI2026}

ในเชิงวิชาการ โมเดลที่เหมาะสมสำหรับการรันบนอุปกรณ์ที่มีทรัพยากรจำกัดคือกลุ่มโมเดลขนาดเล็ก (Small Language Models) เช่น \textbf{Gemma 4 E2B} ซึ่งถูกออกแบบให้มีพารามิเตอร์ที่มีประสิทธิภาพสำหรับการใช้งานระดับ Edge \cite{OllamaGemma4E2B2026} โดยมีความสามารถครอบคลุมทั้งงานด้าน Reasoning, Coding และการทำความเข้าใจสื่อหลายรูปแบบ (Multimodal Understanding) การใช้ LLM ในระบบเฝ้าระวังจึงควรเน้นบทบาทในการเป็นตัวสนับสนุนการตัดสินใจ (Decision Support) ภายใต้กฎเกณฑ์เชิงระบบที่ตรวจสอบได้

เมื่อมีการนำ MCP มาใช้ ระบบจะสามารถแยกส่วนของโมเดลออกจากส่วนของข้อมูลและเครื่องมือได้อย่างชัดเจน ส่งผลดีต่อความปลอดภัยและการควบคุมสิทธิ์การเข้าถึงข้อมูลที่สำคัญ (เช่น ข้อมูลสุขภาพ) โดยที่ตัวโมเดลยังสามารถทำหน้าที่ประมวลผลข้อมูลและเลือกใช้เครื่องมือที่เหมาะสมเพื่อตอบสนองต่อคำร้องขอของผู้ใช้ได้อย่างมีประสิทธิภาพ

\section{ทฤษฎีด้านความปลอดภัยและการควบคุมสิทธิ์การเข้าถึง}
\label{sec:ch2_security_rbac}

ในการพัฒนาระบบเทคโนโลยีสารสนเทศสำหรับสถานดูแลผู้สูงอายุที่มีการจัดการข้อมูลสุขภาพ (Health Information) ข้อมูลส่วนบุคคล (Personally Identifiable Information: PII) และข้อมูลพฤติกรรมที่มีความละเอียดอ่อน การรักษาความปลอดภัยและการจำกัดสิทธิ์การเข้าถึงเป็นเรื่องสำคัญอย่างยิ่ง

\subsection{การควบคุมการเข้าถึงตามบทบาท (Role-Based Access Control: RBAC)}
Role-Based Access Control (RBAC) เป็นโมเดลการควบคุมการเข้าถึงข้อมูลที่กำหนดสิทธิ์ของผู้ใช้ตามบทบาทหน้าที่ในองค์กร แทนที่จะกำหนดสิทธิ์ให้แต่ละบุคคลโดยตรง \cite{Sandhu1996RBAC,NIST2010RBACGuide} โมเดล RBAC ถูกมาตรฐานไอเอสโอรับรองใน ISO/IEC 27002 สำหรับการจัดการความปลอดภัยของสารสนเทศด้านสาธารณสุข \cite{ISO27799_2016} การนำ RBAC มาใช้ในระบบของสถานดูแลผู้สูงอายุ (เช่น ผู้ดูแล, พยาบาลวิชาชีพ, ผู้บริหาร) ช่วยลดความซับซ้อนในการจัดการสิทธิ์ เมื่อบุคลากรมีการเปลี่ยนหน้าที่ ระบบเพียงแค่เปลี่ยนบทบาทของผู้ใช้นั้น ๆ สิทธิ์การเข้าถึงและแก้ไขข้อมูลก็จะปรับตามโดยอัตโนมัติ

\subsection{การรักษาความปลอดภัยของสถาปัตยกรรมระบบ}
นอกจาก RBAC แล้ว การรักษาความลับ (Confidentiality) และความสมบูรณ์ (Integrity) ของข้อมูลระหว่างการส่งผ่านเครือข่ายเป็นสิ่งสำคัญ โดยเฉพาะข้อมูลสุขภาพ (Health Information) ที่ต้องปฏิบัติตามมาตรฐาน HIPAA Security Rule \cite{HHS2013HIPAASecurityRule} การใช้ JSON Web Token (JWT) ตามมาตรฐาน RFC 7519 \cite{RFC7519JWT} ช่วยให้การตรวจสอบสิทธิ์แบบไม่มีสถานะ (Stateless Authentication) เป็นไปอย่างรวดเร็วและปลอดภัย โดยร่วมกับ OAuth 2.0 Authorization Framework \cite{RFC6749OAuth2} สำหรับการมอบสิทธิ์แบบมีขอบเขตจำกัด (Scope Narrowing) นอกจากนี้ การจัดการ Token ให้อยู่ในคุกกี้ที่ทำงานเฉพาะทาง HTTP (HTTP-only Cookie) ช่วยป้องกันการโจมตีแบบ Cross-Site Scripting (XSS) อย่างมีประสิทธิภาพ และลดความเสี่ยงจากการเข้าถึง Token โดยสคริปต์ฝั่งไคลเอ็นต์ที่ไม่ได้รับอนุญาติ

\section{งานวิจัยที่เกี่ยวข้อง}
\label{sec:ch2_related_work}

การทบทวนวรรณกรรมที่ผ่านมาแสดงให้เห็นถึงความพยายามนำเทคโนโลยีมาประยุกต์ใช้เพื่อการดูแลผู้สูงอายุอย่างหลากหลาย งานวิจัยในอดีตมักมุ่งเน้นไปที่การใช้เซ็นเซอร์เพื่อตรวจจับการล้มแบบแยกส่วน โดยขาดการบูรณาการข้อมูลเพื่อสร้างบริบทที่ครอบคลุม (Comprehensive Context) เช่น การนำข้อมูลการเคลื่อนไหว (IMU) มาร่วมกับสัญญาณชีพ (PPG) และพิกัดตำแหน่งภายในอาคาร (IPS) เพื่อวิเคราะห์พฤติกรรมเชิงลึก

นอกจากนี้ งานวิจัยหลายชิ้นที่มีการนำ AI มาใช้ มักประสบปัญหาด้านความเป็นส่วนตัวเนื่องจากการพึ่งพาบริการ Cloud ภายนอก และปัญหาความโปร่งใสของการตัดสินใจ (Black-box AI) ในทางตรงกันข้าม ปริญญานิพนธ์ฉบับนี้ได้นำเสนอสถาปัตยกรรมแบบขอบเครือข่าย (Edge AI) ผ่านการใช้โมเดลภาษาขนาดเล็ก (Gemma 4) ที่ทำงานร่วมกับ Model Context Protocol (MCP) เพื่อจำกัดขอบเขตข้อมูลและเพิ่มความโปร่งใส (Explainability) ของปัญญาประดิษฐ์ในการดูแลผู้ป่วย ส่งผลให้เกิดระบบการดูแลผู้สูงอายุที่ตอบสนองแบบเรียลไทม์ ปลอดภัย และมีประสิทธิภาพสูง
