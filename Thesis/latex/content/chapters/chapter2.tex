\section{บริบทและความท้าทายในการดูแลผู้สูงอายุ}
การเปลี่ยนผ่านสู่สังคมสูงวัยทำให้สถานพักผู้สูงอายุต้องรองรับผู้รับบริการที่มีความซับซ้อนด้านสุขภาพมากขึ้น ทั้งด้านการเคลื่อนไหวและการพึ่งพาอุปกรณ์ช่วยเหลือ (เช่น เก้าอี้รถเข็น) การพึ่งพาการสังเกตด้วยมนุษย์เพียงอย่างเดียวจึงไม่เพียงพอ การดูแลจำเป็นต้องอาศัยข้อมูลเชิงลึก (Insights) แบบเรียลไทม์เพื่อประกอบการตัดสินใจ \cite{WHO2021DecadeAgeing,NSO2021OlderPersonsSurvey}

งานวิจัยชี้ว่าบุคลากรทางการแพทย์ต้องเผชิญกับความเครียดและ "ภาวะเหนื่อยล้าจากการแจ้งเตือน" (Alert Fatigue) หากระบบแจ้งเตือนไม่มีการคัดกรองบริบทที่ดีพอ ดังนั้นการออกแบบระบบติดตามจึงต้องมุ่งเน้นที่ความแม่นยำและลดภาระทางปัญญา (Cognitive Load) ของผู้ใช้งาน \cite{Gates2018AlertFatigue}

\section{เทคโนโลยีอินเทอร์เน็ตของสรรพสิ่งและเครือข่ายเซ็นเซอร์}
\subsection{สถาปัตยกรรม IoT และเซ็นเซอร์สวมใส่}
อินเทอร์เน็ตของสรรพสิ่ง (Internet of Things: IoT) เป็นสถาปัตยกรรมที่ช่วยเชื่อมต่ออุปกรณ์กายภาพ (เช่น เซ็นเซอร์ IMU, PPG, และตำแหน่ง) สู่ระบบประมวลผลส่วนกลาง การดูแลผู้สูงอายุแบบ AAL (Ambient Assisted Living) อาศัย IoT เพื่อส่งเสริมความอิสระและติดตามความปลอดภัยแบบ 24/7 \cite{Atzori2010IoTSurvey}

\plannedchtwofigure{fig:ch2_detailed_iot_architecture}{iot_architecture.png}{0.8}{ภาพสถาปัตยกรรม IoT แบบหลายชั้นสำหรับงาน AAL}{สถาปัตยกรรม IoT แบบละเอียด 5 ชั้นสำหรับระบบดูแลผู้สูงอายุ}

ในการประมวลผลข้อมูลจากเซ็นเซอร์ (Sensor Fusion) โดยเฉพาะสัญญาณ IMU (ความเร่งและการหมุน) และสัญญาณ PPG (อัตราการเต้นของหัวใจ) อัลกอริทึมเช่น Extended Kalman Filter นิยมถูกนำมาใช้เพื่อลดสัญญาณรบกวนและเพิ่มความเสถียรของฟีเจอร์ที่ได้ \cite{Welch2006KalmanFilter}

\subsection{การระบุตำแหน่งภายในอาคาร (IPS) และการส่งข้อมูลใกล้เวลาจริง}
ระบบระบุตำแหน่งภายในอาคาร (Indoor Positioning System: IPS) ที่ใช้สัญญาณ Bluetooth Low Energy (BLE) ผ่านกระบวนการ Fingerprinting อาศัยโมเดลเพื่อนบ้านใกล้สุด (k-NN) หรือแบบจำลองต้นไม้ตัดสินใจร่วม (Random Forest) เพื่อจำแนกตำแหน่ง (Room-level) จากค่าความแรงสัญญาณ (RSSI)

การส่งข้อมูลเซ็นเซอร์เหล่านี้อย่างต่อเนื่อง ต้องการโพรโทคอลที่ใช้ทรัพยากรน้อยและรองรับข้อจำกัดแบนด์วิดท์ เช่น MQTT (Message Queuing Telemetry Transport) ซึ่งทำงานแบบ Publish-Subscribe การเลือกระดับคุณภาพบริการ (QoS 0 สำหรับค่าต่อเนื่อง, QoS 1 สำหรับแจ้งเตือนฉุกเฉิน) ช่วยจัดการคิวข้อความและป้องกันภาวะคอขวด (Back-pressure) ของเครือข่าย \cite{OASIS2019MQTTv5,Kleppmann2017_DDIA}

\plannedchtwofigure{fig:ch2_mqtt_flow}{mqtt_qos.png}{0.75}{ภาพแสดงการเปรียบเทียบระดับ QoS ของโพรโทคอล MQTT}{ลำดับการรับส่งข้อความในโพรโทคอล MQTT ตามระดับ QoS}

\section{ปัญญาประดิษฐ์และสถาปัตยกรรมแบบจำลองภาษา}
การประมวลผลบริบทซับซ้อนในยุคปัจจุบันสามารถทำได้ด้วยแบบจำลองภาษาขนาดใหญ่ (Large Language Models: LLM) ที่ทำงานบนสถาปัตยกรรม Transformer อย่างไรก็ตาม การประยุกต์ใช้ในการแพทย์และการดูแลผู้สูงอายุต้องการความแม่นยำและกลไกป้องกันการสร้างข้อมูลเท็จ (Hallucination)

\plannedchtwofigure{fig:ch2_llm_mcp}{llm_mcp.png}{0.85}{ภาพอธิบายการเชื่อมต่อ LLM กับบริบทภายนอกผ่าน MCP}{สถาปัตยกรรมการเชื่อมต่อ LLM กับเครื่องมือภายนอกผ่าน Model Context Protocol (MCP)}

เทคโนโลยี Model Context Protocol (MCP) ถูกพัฒนาขึ้นเพื่อแก้ปัญหาข้อจำกัดด้านบริบทของ LLM โดยเป็นมาตรฐานในการเชื่อมโยงโมเดลเข้ากับ "เครื่องมือ" (Tools) และ "ทรัพยากร" (Resources) ในระบบขององค์กรอย่างปลอดภัยในรูปแบบ Propose-Confirm-Execute เอเจนต์จึงสามารถดึงข้อมูลแบบเรียลไทม์ และสรุปผลหรือสั่งการได้โดยยังคงมี "มนุษย์ในลูป" (Human-in-the-loop) กำกับดูแลอยู่เสมอ \cite{Anthropic2024MCP,Yao2023ReAct}

\section{ความปลอดภัยสารสนเทศและการประเมินระบบ}
ระบบสารสนเทศสุขภาพต้องปฏิบัติตามมาตรฐานความเป็นส่วนตัวและกฎหมายคุ้มครองข้อมูล (เช่น PDPA) การใช้สถาปัตยกรรม "การควบคุมการเข้าถึงตามบทบาท" (Role-Based Access Control: RBAC) ร่วมกับการยืนยันตัวตนแบบ JWT ช่วยรับประกันหลักการเข้าถึงเท่าที่จำเป็น (Least Privilege) \cite{PDPA2019Thailand,Sandhu1996RBAC}

สำหรับการประเมินประสบการณ์ผู้ใช้ (User Experience) นิยมใช้แบบประเมินมาตรฐานอย่าง System Usability Scale (SUS) และแบบสอบถาม CSUQ เพื่อให้สามารถวัดผลเชิงปริมาณและสถิติได้อย่างน่าเชื่อถือ ซึ่งสถิติเหล่านี้สะท้อนถึงการลดภาระความยุ่งยากของบุคลากรได้อย่างเป็นรูปธรรม \cite{Brooke1996SUS}

\plannedchtwofigure{fig:ch2_security_ux}{security_ux.png}{0.8}{ภาพแสดงแนวคิด RBAC และมาตรวัดความพึงพอใจ SUS}{องค์ประกอบความปลอดภัยของข้อมูลและการประเมินความพึงพอใจของผู้ใช้}

\section{งานวิจัยที่เกี่ยวข้องและช่องว่างการวิจัย}
จากการทบทวนวรรณกรรมพบว่า แม้จะมีงานวิจัยด้านการระบุตำแหน่ง (IPS) หรืออุปกรณ์ช่วยติดตามกิจกรรมผู้สูงอายุอยู่เป็นจำนวนมาก แต่ระบบส่วนใหญ่ยังมีลักษณะแยกส่วน ขาดการบูรณาการข้อมูลเข้าสู่ลานปฏิบัติงาน (Workflow) ของพยาบาลแบบศูนย์รวม และยังขาดการประยุกต์ใช้ผู้ช่วยอัจฉริยะ (LLM) ในการให้ข้อมูลสรุปเชิงบริบทที่ผูกกับระบบจริงได้อย่างปลอดภัย ดังตารางที่~\ref{tab:ch2_related_work_summary}
