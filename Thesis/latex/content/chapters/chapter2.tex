\section{บริบทการดูแลผู้สูงอายุและความท้าทายในสถานดูแล}
\label{sec:ch2_care_context}

การเปลี่ยนผ่านสู่สังคมสูงวัย (Aging Society) ส่งผลกระทบโดยตรงต่อระบบการจัดการในสถานดูแลผู้สูงอายุ (Nursing Homes หรือ Care Facilities) ซึ่งต้องรองรับผู้รับบริการที่มีความเปราะบางและมีความต้องการที่หลากหลายมากขึ้น \cite{WHO2021DecadeAgeing} ในบริบทของสถานดูแลขนาดกลางถึงใหญ่ เช่น บ้านพักคนชราวาสนะเวศม์ ซึ่งมีพื้นที่กว้างขวางและมีอาคารพักอาศัยหลายหลัง ความท้าทายที่สำคัญคืออัตราส่วนระหว่างบุคลากรและผู้สูงอายุ โดยในขณะที่สถานดูแลระดับพรีเมียมอาจมีอัตราส่วนที่ 1:1 ถึง 1:4 แต่ในสถานดูแลของรัฐหรือสถานดูแลขนาดใหญ่ทั่วไป อัตราส่วนผู้ดูแลต่อผู้สูงอายุอาจสูงถึง 1:20 โดยเฉพาะในช่วงเวรกลางคืน \cite{Schulz2020CaregiverBurden, SituationThaiElderly2022} ซึ่งเป็นตัวเลขที่สะท้อนถึงวิกฤตการขาดแคลนแรงงานในภาคส่วนการดูแลผู้สูงอายุทั้งในระดับประเทศและระดับโลก การมีผู้ดูแลเพียง 1 คนต่อผู้รับบริการ 20 ท่าน ทำให้เกิด "ช่องว่างในการเฝ้าระวัง" (Monitoring Gap) ที่รุนแรง และความเสี่ยงต่อการเกิดอุบัติเหตุหรือภาวะฉุกเฉินที่ไม่ได้รับการตอบสนองทันท่วงที

แนวคิด \textbf{Ambient Assisted Living (AAL)} คือการบูรณาการเทคโนโลยี \textbf{Internet of Things (IoT)} และ \textbf{Smart Environment (สภาพแวดล้อมอัจฉริยะ)} เข้าด้วยกัน เพื่อสร้างระบบสนับสนุนการใช้ชีวิตของผู้สูงอายุ โดย IoT ทำหน้าที่เป็นโครงสร้างพื้นฐานในการตรวจวัดและสื่อสารข้อมูลจากอุปกรณ์สวมใส่ \cite{Atzori2010IoTSurvey} ในขณะที่สภาพแวดล้อมอัจฉริยะจะช่วยประมวลผลให้พื้นที่นั้นสามารถรับรู้และตอบสนองต่อพฤติกรรมของมนุษย์ได้อย่างแนบเนียน \cite{Rashidi2013AALSurvey} การผสานรวมกันนี้ทำให้ระบบ AAL สามารถมุ่งเน้นที่ความปลอดภัย ความเป็นอิสระ และการเฝ้าระวังแบบไม่ล่วงละเมิดความเป็นส่วนตัว (Non-intrusive) ซึ่งถือเป็นหัวใจสำคัญในการเปลี่ยนผ่านจากการดูแลเชิงรับ (Reactive Care) ไปสู่การดูแลเชิงรุก (Proactive Care) ที่ตอบสนองต่อเหตุฉุกเฉินหรือความผิดปกติได้อย่างทันท่วงที

\section{ทฤษฎีระบบระบุตำแหน่งภายในอาคาร}
\label{sec:ch2_ips}

เนื่องจากระบบระบุตำแหน่งผ่านดาวเทียม (GPS) มีข้อจำกัดในการใช้งานภายในอาคารเนื่องจากสัญญาณถูกบดบังด้วยโครงสร้างอาคาร ระบบระบุตำแหน่งภายในอาคาร (Indoor Positioning System: IPS) จึงถูกพัฒนาขึ้นเพื่อระบุพิกัดในพื้นที่ปิด เทคนิคการระบุตำแหน่งพื้นฐานที่แพร่หลายมีดังนี้:

\subsection{เทคนิคเชิงระยะทาง (Range-based Techniques)}
อาศัยการคำนวณระยะทางระหว่างตัวรับและตัวส่งสัญญาณ โดยเทคนิคที่แพร่หลายที่สุดคือ \textbf{Trilateration} ซึ่งคำนวณจุดตัดของวงกลม 3 วงที่เกิดจากจุดอ้างอิงอ้างอิง 3 จุด ดังรูปที่~\ref{fig:ch2_trilateration}

\begin{figure}[!ht]
    \centering
    \begin{tikzpicture}[scale=0.6]
        % Define coordinates for 3 anchors
        \coordinate (A1) at (0,0);
        \coordinate (A2) at (5,1);
        \coordinate (A3) at (2,4);
        
        % Define radii
        \def\rOne{3}
        \def\rTwo{2.5}
        \def\rThree{2.1}
        
        % Intersection point (approximate for illustration)
        \coordinate (P) at (2.4, 1.8);
        
        % Draw circles
        \draw[thick, blue!50] (A1) circle (\rOne);
        \draw[thick, red!50] (A2) circle (\rTwo);
        \draw[thick, green!50] (A3) circle (\rThree);
        
        % Draw anchors
        \filldraw (A1) circle (2pt) node[below left] {$A_1(x_1, y_1)$};
        \filldraw (A2) circle (2pt) node[right] {$A_2(x_2, y_2)$};
        \filldraw (A3) circle (2pt) node[above] {$A_3(x_3, y_3)$};
        
        % Draw target point P
        \filldraw[red] (P) circle (3pt) node[below right, font=\bfseries] {$P(x, y)$};

        \draw[dashed, gray] (A1) -- (P) node[midway, above left, black] {$d_1$};
        \draw[dashed, gray] (A2) -- (P) node[midway, above right, black] {$d_2$};
        \draw[dashed, gray] (A3) -- (P) node[midway, left, black] {$d_3$};
        
    \end{tikzpicture}
    \caption{หลักการระบุตำแหน่งแบบ Trilateration จากจุดอ้างอิง 3 จุด}
    \label{fig:ch2_trilateration}
\end{figure}

หลักการของ Trilateration คือการหาจุดตัดของวงกลม 3 วง โดยพิกัดเป้าหมาย $(x, y)$ สามารถคำนวณได้จากความสัมพันธ์ระหว่างพิกัดจุดอ้างอิง $(x_i, y_i)$ และระยะทาง $d_i$ ดังสมการ:

\begin{equation}
\text{พิกัด } P(x, y) \text{ คำนวณได้จากระบบสมการ:} \quad
\begin{cases}
(x - x_1)^2 + (y - y_1)^2 = d_1^2 \\
(x - x_2)^2 + (y - y_2)^2 = d_2^2 \\
(x - x_3)^2 + (y - y_3)^2 = d_3^2 
\end{cases}
\label{eq:ch2_trilateration_sys}
\end{equation}

เมื่อ $(x_i, y_i)$ คือพิกัดของจุดอ้างอิงที่ $i$ และ $d_i$ คือระยะทางที่วัดได้จากจุดอ้างอิงนั้นๆ การแก้สมการเพื่อหาพิกัด $(x, y)$ สามารถทำได้โดยการนำสมการที่ 1 ไปลบออกจากสมการที่ 2 และ 3 เพื่อกำจัดตัวแปรยกกำลังสอง จะได้ระบบสมการเชิงเส้นในรูปแบบเมทริกซ์ $AX = B$ ดังสมการ~(\ref{eq:ch2_trilateration_matrix}):

\begin{equation}
\begin{bmatrix}
2(x_2 - x_1) & 2(y_2 - y_1) \\
2(x_3 - x_1) & 2(y_3 - y_1)
\end{bmatrix}
\begin{bmatrix}
x \\
y
\end{bmatrix}
=
\begin{bmatrix}
(x_2^2 - x_1^2) + (y_2^2 - y_1^2) + (d_1^2 - d_2^2) \\
(x_3^2 - x_1^2) + (y_3^2 - y_1^2) + (d_1^2 - d_3^2)
\end{bmatrix}
\label{eq:ch2_trilateration_matrix}
\end{equation}

ซึ่งพิกัดเป้าหมาย $P(x, y)$ จะคำนวณได้จากการหาอินเวอร์สของเมทริกซ์ $A$ ต่อไป

สำหรับการหาระยะทาง ($d$) จากความแรงของสัญญาณวิทยุ มักใช้แบบจำลองการลดทอนของสัญญาณ (Log-distance Path Loss Model) ดังรูปที่~\ref{fig:ch2_rssi_dist} และสมการ~(\ref{eq:ch2_pathloss}):

\begin{figure}[!ht]
    \centering
    \begin{tikzpicture}[scale=1.1]
        % Axes
        \draw[->] (0,-4) -- (6,-4) node[right] {ระยะทาง $d$ (เมตร)};
        \draw[->] (0,-4) -- (0,0.5) node[above] {RSSI (dBm)};
        
        % Curve
        \draw[thick, blue, domain=0.5:5.5, smooth, variable=\x] plot ({\x}, {-1 - 1.5*log10(\x)});
        
        \node[blue, right] at (2.5,-2.2) {$\mathrm{RSSI}(d) = \mathrm{RSSI}(d_0) - 10n \log_{10}(d)$};
        
        % Reference point at 1m
        \filldraw (1,-1) circle (1.5pt);
        \draw[dashed] (1,-4) -- (1,-1) node[midway, left, font=\scriptsize] {$1$ ม.};
        \draw[dashed] (0,-1) -- (1,-1) node[left] {$\mathrm{RSSI}(d_0)$};
    \end{tikzpicture}
    \caption{ความสัมพันธ์ระหว่างค่า RSSI และระยะทางตามแบบจำลอง Path Loss}
    \label{fig:ch2_rssi_dist}
\end{figure}

\begin{equation}
\mathrm{RSSI}(d) = \mathrm{RSSI}(d_0) - 10n \log_{10}\left(\frac{d}{d_0}\right) + X_{\sigma}
\label{eq:ch2_pathloss}
\end{equation}
เมื่อ $\mathrm{RSSI}(d)$ คือกำลังสัญญาณที่ได้รับที่ระยะ $d$, $\mathrm{RSSI}(d_0)$ คือกำลังสัญญาณอ้างอิงที่ระยะ 1 เมตร, $n$ คือค่าสัมประสิทธิ์การสูญเสียสัญญาณในพื้นที่ และ $X_{\sigma}$ คือสัญญาณรบกวนที่มีการกระจายตัวแบบปกติ \cite{Kaemarungsi2012RSSIAnalysis}

\subsection{เทคนิคการสร้างแผนที่สัญญาณ (Fingerprinting)}
เป็นวิธีที่นิยมใช้ในอาคารที่มีโครงสร้างซับซ้อน แบ่งเป็น 2 ขั้นตอนหลัก ดังแสดงกระบวนการในรูปที่~\ref{fig:ch2_fingerprinting}:
\begin{enumerate}
    \item \textbf{Offline Phase:} เก็บข้อมูลความแรงสัญญาณ (RSSI) จากตัวส่งสัญญาณทุกจุดในแต่ละพิกัดพื้นที่ เพื่อสร้างเป็น "ลายนิ้วมือสัญญาณ" (Radio Map)
    \item \textbf{Online Phase:} นำสัญญาณที่วัดได้จริงเปรียบเทียบกับ Radio Map โดยใช้อัลกอริทึม เช่น K-Nearest Neighbors (KNN) เพื่อหาตำแหน่งที่ใกล้เคียงที่สุด
\end{enumerate}

สำหรับการคำนวณความใกล้เคียงระหว่างสัญญาณที่วัดได้แบบเรียลไทม์กับลายนิ้วมือสัญญาณในฐานข้อมูล มักใช้การหาระยะห่างแบบยูคลิด (Euclidean Distance) ดังสมการ:
\begin{equation}
D_j = \sqrt{\sum_{i=1}^{N} (\mathrm{RSSI}_{i} - \mathrm{RSSI}_{i,j})^2}
\label{eq:ch2_knn_distance}
\end{equation}
เมื่อ $D_j$ คือระยะห่างแบบยูคลิดระหว่างสัญญาณที่วัดได้กับลายนิ้วมือสัญญาณที่จุดอ้างอิง $j$, $N$ คือจำนวนจุดส่งสัญญาณ (Beacon) ทั้งหมดที่ครอบคลุม, $\mathrm{RSSI}_{i}$ คือค่าความแรงสัญญาณจาก Beacon ตัวที่ $i$ ที่วัดได้ในขณะนั้น และ $\mathrm{RSSI}_{i,j}$ คือค่าความแรงสัญญาณจาก Beacon ตัวที่ $i$ ณ จุดอ้างอิง $j$ ในฐานข้อมูล โดยระบบจะเลือกตำแหน่งอ้างอิงที่มีค่า $D_j$ ต่ำที่สุด หรือเฉลี่ยจาก $K$ จุดที่ต่ำที่สุด เพื่อใช้เป็นพิกัดเป้าหมาย

        % Online Phase
        \node[draw, rectangle, rounded corners, fill=green!10, text width=3cm, align=center, right=of offline] (online) {\textbf{Online Phase}\\เปรียบเทียบสัญญาณ\\(Pattern Matching)};

\section{เทคโนโลยีเซ็นเซอร์สำหรับการตรวจวัดกิจกรรม}
\label{sec:ch2_sensors}

เทคโนโลยีเซ็นเซอร์แบบสวมใส่เป็นองค์ประกอบสำคัญในระบบติดตามและดูแลสุขภาพ แบ่งออกเป็นเซ็นเซอร์ทางสรีรวิทยา (Physiological Sensors) และเซ็นเซอร์ทางกายภาพ (Inertial Sensors)

\subsection{Photoplethysmography (PPG)}

เซ็นเซอร์ PPG ตรวจวัดการเปลี่ยนแปลงของปริมาตรเลือดในหลอดเลือดฝอยด้วยการฉายแสงและวัดการสะท้อน ดังแสดงในรูปที่~\ref{fig:ch2_ppg_principle} ข้อมูลหลักที่ได้คืออนุกรมเวลาของช่วงระหว่างจังหวะหัวใจ (RR-interval) ซึ่งนำไปคำนวณค่าต่างๆ ดังนี้

\textbf{(1) อัตราการเต้นของหัวใจ (Heart Rate):}
\begin{equation}
\mathrm{HR} = \frac{60}{\overline{RR}}
\label{eq:ch2_hr}
\end{equation}
เมื่อ $\overline{RR}$ คือค่าเฉลี่ยของช่วง RR-interval (วินาที) และ $\mathrm{HR}$ มีหน่วยเป็น bpm (ครั้งต่อนาที)

\textbf{(2) ความแปรปรวนของอัตราการเต้นของหัวใจ (HRV) ด้วย SDNN:}
\begin{equation}
\mathrm{SDNN} = \sqrt{\frac{1}{N-1}\sum_{i=1}^{N}(RR_i - \overline{RR})^2}
\label{eq:ch2_sdnn}
\end{equation}
เมื่อ $N$ คือจำนวนช่วง RR ทั้งหมด และ $RR_i$ คือช่วง RR ที่ $i$ ค่า SDNN ที่ต่ำลงบ่งบอกถึงระบบประสาทอัตโนมัติที่ตอบสนองน้อยลง

\textbf{(3) RMSSD สำหรับประเมินกิจกรรมประสาทพาราซิมพาเทติก:}
\begin{equation}
\mathrm{RMSSD} = \sqrt{\frac{1}{N-1}\sum_{i=1}^{N-1}(RR_{i+1} - RR_i)^2}
\label{eq:ch2_rmssd}
\end{equation}
ค่า RMSSD นิยมใช้ตรวจจับความเครียดทางกายภาพ (Physical Stress) เพราะสะท้อนการควบคุมอัตราการเต้นของหัวใจในช่วงสั้น

\subsection{Inertial Measurement Unit (IMU)}

เซ็นเซอร์ IMU ประกอบด้วย Accelerometer วัดความเร่งในสามแกน $(a_x, a_y, a_z)$ และ Gyroscope วัดอัตราการหมุน $(\omega_x, \omega_y, \omega_z)$ ในระบบที่ติดตั้งอุปกรณ์บนล้อเก้าอี้รถเข็น ดังรูปที่~\ref{fig:ch2_imu_wheel} การคำนวณระยะทางอาศัย \textbf{Gyroscope} แกน Z เป็นหลัก เพื่อวัดอัตราการหมุนของล้อและแปลงเป็นการเคลื่อนที่เชิงเส้นผ่านรัศมีล้อ

\textbf{(1) การลด Noise จากแรงโน้มถ่วงด้วย EMA Low-pass Filter บน Gyroscope:}

เนื่องจาก Gyroscope มีสัญญาณรบกวน (Sensor Noise) และค่าออฟเซตคงที่ (Zero-rate offset) จึงใช้ Exponential Moving Average (EMA) เพื่อลด Noise ก่อนนำไปคำนวณ:
\begin{equation}
\tilde{\omega}_{z,k} = \alpha \cdot \omega_{z,k} + (1 - \alpha) \cdot \tilde{\omega}_{z,k-1}
\label{eq:ch2_ema_lpf}
\end{equation}
เมื่อ $\tilde{\omega}_{z,k}$ คืออัตราการหมุนหลังกรองที่เวลา $k$, $\omega_{z,k}$ คือค่าดิบจาก Gyroscope Z, และ $\alpha$ คือค่าคงที่ของ Filter ($0 < \alpha < 1$) นอกจากนี้ยังใช้ Deadband กำจัดสัญญาณรบกวนขนาดเล็ก:
\begin{equation}
\hat{\omega}_{z,k} =
\begin{cases}
\tilde{\omega}_{z,k} & \text{ถ้า } |\tilde{\omega}_{z,k}| \geq \omega_{\mathrm{dead}} \\
0 & \text{อื่นๆ}
\end{cases}
\label{eq:ch2_deadband}
\end{equation}
เมื่อ $\omega_{\mathrm{dead}}$ คือค่า Deadband threshold (หน่วย dps)

\textbf{(2) การคำนวณมุมการหมุนของล้อ:}
\begin{equation}
\Delta\theta_k = \hat{\omega}_{z,k} \cdot \frac{\pi}{180} \cdot \Delta t
\label{eq:ch2_angular_delta}
\end{equation}
เมื่อ $\Delta\theta_k$ คือมุมการหมุน (เรเดียน) ในช่วงเวลา $\Delta t$ และการคูณด้วย $\pi/180$ เพื่อแปลงจาก degree/s เป็น rad/s

\textbf{(3) การคำนวณระยะทางผ่านรัศมีล้อ (Wheel Odometry):}
\begin{equation}
\Delta s_k = |\Delta\theta_k| \cdot r
\label{eq:ch2_arc_length}
\end{equation}
\begin{equation}
s_k = s_{k-1} + \Delta s_k
\label{eq:ch2_odometry}
\end{equation}
เมื่อ $r$ คือรัศมีของล้อเก้าอี้รถเข็น (เมตร) และ $\Delta s_k$ คือระยะทางที่เคลื่อนที่ในช่วงเวลา $\Delta t$ วิธีนี้เรียกว่า \textbf{Wheel Odometry} ซึ่งให้ความแม่นยำสูงกว่าการอินทิเกรต Accelerometer โดยตรง เนื่องจากหลีกเลี่ยงปัญหา Drift สะสม

\textbf{(4) การคำนวณความเร็วและความเร่งเชิงเส้น:}
\begin{equation}
v_k = \frac{s_{\mathrm{win}}}{T_{\mathrm{win}}}
\label{eq:ch2_velocity_win}
\end{equation}
\begin{equation}
a_k = \frac{v_k - v_{k-1}}{T_{\mathrm{win}}}
\label{eq:ch2_accel_win}
\end{equation}
เมื่อ $s_{\mathrm{win}}$ คือระยะทางสะสมในหน้าต่างเวลา $T_{\mathrm{win}}$ (Sliding Window) วิธีนี้ช่วยลดผลกระทบจาก Noise ในการประมาณค่าความเร็วและความเร่ง

\textbf{(5) การหาทิศทางการเคลื่อนที่:}
\begin{equation}
\mathrm{direction}_k =
\begin{cases}
+1 & \text{ถ้า } \hat{\omega}_{z,k} > 0 \quad \text{(เดินหน้า)} \\
-1 & \text{ถ้า } \hat{\omega}_{z,k} < 0 \quad \text{(ถอยหลัง)} \\
0  & \text{ถ้า } \hat{\omega}_{z,k} = 0 \quad \text{(หยุดนิ่ง)}
\end{cases}
\label{eq:ch2_direction}
\end{equation}

\textbf{(6) การหาทิศทางการเอียง (Orientation) จาก Accelerometer:}

Accelerometer ใช้สำหรับคำนวณมุมเอียงของตัวอุปกรณ์ (Roll/Pitch) เพื่อประเมินสถานะของผู้ใช้งาน:
\begin{equation}
\phi = \arctan\!\left(\frac{a_y}{a_z}\right), \quad
\theta = \arctan\!\left(\frac{-a_x}{\sqrt{a_y^2 + a_z^2}}\right)
\label{eq:ch2_roll_pitch}
\end{equation}
เมื่อ $\phi$ คือมุม Roll และ $\theta$ คือมุม Pitch ซึ่งสามารถนำมาใช้ตรวจจับการเอียงผิดปกติที่อาจบ่งบอกถึงการล้มหรือภาวะฉุกเฉินได้




\section{เทคโนโลยีซอฟต์แวร์และโครงสร้างพื้นฐาน}
\label{sec:ch2_software_infra}

\subsection{กรอบงานเว็บเซิร์ฟเวอร์และการจัดการฐานข้อมูล}
\textbf{FastAPI} เป็นกรอบงาน (Framework) สำหรับสร้าง REST API ด้วยภาษา Python ที่เน้นความเร็วและประสิทธิภาพสูง รองรับการทำงานแบบอะซิงโครนัส (Asynchronous) และใช้ Pydantic สำหรับตรวจสอบโครงสร้างข้อมูล (Data Validation) อัตโนมัติ รวมถึงมีระบบ Dependency Injection ที่ช่วยให้จัดการส่วนประกอบต่างๆ ได้อย่างยืดหยุ่น 
สำหรับการจัดการฐานข้อมูล การใช้ \textbf{SQLAlchemy} ซึ่งเป็น Object-Relational Mapping (ORM) ช่วยแปลงข้อมูลระหว่างฐานข้อมูลเชิงสัมพันธ์และออบเจ็กต์ในโค้ด โดยทำงานร่วมกับ \textbf{Alembic} เพื่อจัดการการเปลี่ยนแปลงโครงสร้างฐานข้อมูล (Database Migrations)
\textbf{PostgreSQL} เป็นระบบจัดการฐานข้อมูลเชิงสัมพันธ์ที่มีความเสถียรสูง รองรับหลักการ ACID (Atomicity, Consistency, Isolation, Durability) และสามารถจัดการข้อมูลกึ่งโครงสร้าง (Semi-structured data) ผ่านประเภทข้อมูล JSONB ได้อย่างมีประสิทธิภาพ

\subsection{กรอบงานส่วนหน้าและการจัดการสถานะ (Frontend Framework)}
\textbf{Next.js} เป็น React Framework สำหรับพัฒนาเว็บแอปพลิเคชันแบบ Full-stack รองรับระบบเส้นทางตามหน้า (App Router) และ React Server Components (RSC) ซึ่งช่วยลดภาระการประมวลผลฝั่งไคลเอนต์ 
สำหรับการจัดการข้อมูลสถานะฝั่งไคลเอนต์ (Client State Management) \textbf{TanStack Query} เป็นไลบรารีที่ช่วยจัดการการแคช (Caching) และการดึงข้อมูลเบื้องหลัง (Background Refetching) 
ด้านความปลอดภัย การจัดการเซสชันนิยมใช้ \textbf{JSON Web Token (JWT)} ร่วมกับ HttpOnly Cookie เพื่อป้องกันการโจมตีแบบ Cross-Site Scripting (XSS) และใช้หลักการ \textbf{Role-Based Access Control (RBAC)} เพื่อจำกัดสิทธิ์การเข้าถึงทรัพยากรตามบทบาทของผู้ใช้งาน

\subsection{แบบจำลองภาษาขนาดใหญ่ (Large Language Models) และ Ollama}
\textbf{แบบจำลองภาษาขนาดใหญ่ (LLM)} อาศัยสถาปัตยกรรม Transformer ซึ่งใช้กลไกการให้ความสนใจในตนเอง (Self-attention) ในการประมวลผลความสัมพันธ์ของคำในประโยค ดังสมการ~(\ref{eq:ch2_attention}):
\begin{equation}
\mathrm{Attention}(Q, K, V) = \mathrm{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V
\label{eq:ch2_attention}
\end{equation}
เมื่อ $Q$ (Query), $K$ (Key), และ $V$ (Value) คือเมทริกซ์ที่ได้จากการแปลงข้อมูลนำเข้า และ $d_k$ คือมิติของ Key การใช้ Multi-head Attention ช่วยให้ระบบสามารถจับความสัมพันธ์ที่หลากหลายได้พร้อมกัน
\textbf{Ollama} เป็นรันไทม์ (Runtime) สำหรับประมวลผล LLM ภายในเครื่อง (On-premise) ซึ่งช่วยจัดการโมเดล การลดขนาดพารามิเตอร์ (Quantization) และให้บริการ API ที่เข้ากันได้กับมาตรฐานสากล ทำให้สามารถใช้งาน AI บนฮาร์ดแวร์ระดับขอบ (Edge Devices) ได้อย่างมีประสิทธิภาพ

\subsection{Model Context Protocol (MCP) และการจำแนกเจตนา}
\textbf{Model Context Protocol (MCP)} เป็นมาตรฐานเปิดสำหรับเชื่อมต่อผู้ช่วย AI เข้ากับแหล่งข้อมูลและเครื่องมือภายนอก ประกอบด้วยองค์ประกอบหลัก 3 ส่วน ได้แก่ เครื่องมือ (Tools), เทมเพลตคำสั่ง (Prompts), และทรัพยากรข้อมูล (Resources) 
ในการทำงานของ AI Agent การทำความเข้าใจเจตนาของผู้ใช้ (Intent Classification) มักอาศัยการวัดความคล้ายคลึงของเวกเตอร์ความหมาย เช่น การใช้ Cosine Similarity ดังสมการ~(\ref{eq:ch2_cosine}):
\begin{equation}
\mathrm{sim}(u, v) = \frac{u \cdot v}{\|u\| \|v\|}
\label{eq:ch2_cosine}
\end{equation}
เมื่อ $u$ และ $v$ คือเวกเตอร์ตัวแทนของข้อความ เพื่อเปรียบเทียบว่าคำสั่งนั้นใกล้เคียงกับเจตนาใดมากที่สุด

\subsection{เทคโนโลยีคอนเทนเนอร์และโปรโตคอล IoT}
\textbf{Docker} เป็นแพลตฟอร์มสำหรับสร้างและจัดการคอนเทนเนอร์ (Containerization) ช่วยแยกบริการต่างๆ ออกเป็นส่วนย่อย (Microservices) ทำให้สามารถติดตั้งและขยายระบบได้โดยไม่ผูกติดกับสภาพแวดล้อมของระบบปฏิบัติการ
\textbf{MQTT (Message Queuing Telemetry Transport)} เป็นโปรโตคอลการสื่อสารแบบ Publish/Subscribe ที่มีนํ้าหนักเบา ออกแบบมาสำหรับอุปกรณ์ IoT ที่มีข้อจำกัดด้านทรัพยากรและแบนด์วิดท์ โดยมีระดับคุณภาพการบริการ (Quality of Service: QoS) 3 ระดับ ได้แก่ QoS 0 (ส่งครั้งเดียวและไม่รอยืนยัน), QoS 1 (รอยืนยันและส่งซ้ำหากไม่ได้รับ), และ QoS 2 (ยืนยันแบบ 4-way handshake เพื่อให้มั่นใจว่าข้อมูลไม่สูญหายและไม่ซ้ำซ้อน)

\subsection{เทคโนโลยีไมโครคอนโทรลเลอร์และฮาร์ดแวร์ระดับขอบ}
การทำงานของระบบ IoT และระบบระบุตำแหน่ง จำเป็นต้องอาศัยฮาร์ดแวร์แบบฝังตัวเพื่อประมวลผลและสื่อสารข้อมูล:
\begin{itemize}
    \item \textbf{ไมโครคอนโทรลเลอร์ (Microcontroller):} ทำหน้าที่เป็นสมองกลฝังตัวสำหรับอ่านค่าจากเซ็นเซอร์ (เช่น IMU) และส่งข้อมูลผ่านเครือข่ายไร้สาย (Wi-Fi หรือ BLE)
    \item \textbf{คอมพิวเตอร์ระดับขอบ (Edge Server):} เป็นคอมพิวเตอร์ขนาดเล็กที่มีประสิทธิภาพสูงกว่าไมโครคอนโทรลเลอร์ ทำหน้าที่ประมวลผลข้อมูลเบื้องต้น จัดการการเชื่อมต่อเครือข่าย และรองรับการทำงานของคอนเทนเนอร์ เพื่อลดภาระการส่งข้อมูลไปยังคลาวด์ส่วนกลาง
\end{itemize}
