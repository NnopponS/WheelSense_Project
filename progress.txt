## Invariant (do not violate)
- Keep `proxy.ts` role guards and FastAPI workspace scoping intact.
- No new backend routes unless P6. Reuse existing contracts in `server/app/api/**`.
- Every i18n key MUST have en + th. Build will fail otherwise (`frontend/lib/i18n.tsx`).
- `Patient.room_id` is canonical for facility placement. Don't invent alt sources.
- Simulator seed must be idempotent (check existing before insert).
- All `run_command` on Windows: NO `cd`, use `Cwd`. Use `python` not `python3`.
- **DOCKER-FIRST for tests and runtime.** This project is primarily run via Docker Compose. Do NOT invoke host-local `python -m pytest` or `python scripts/*.py` when verifying server-side changes. Always route through the compose services.
  - Backend tests:
    `docker compose -f docker-compose.sim.yml exec wheelsense-platform-server python -m pytest tests/ -q`
  - Seed / scripts:
    `docker compose -f docker-compose.sim.yml exec wheelsense-platform-server python scripts/seed_redesign_demo.py --reset`
  - Alembic migrations:
    `docker compose -f docker-compose.sim.yml exec wheelsense-platform-server alembic upgrade heads`
  - MQTT simulator lives in the `wheelsense-simulator` service (sim entry only).
  - Frontend build stays host-local (`npm run build`) because the Next.js container rebuild cycle is slower than the type-check we actually need.
  - Only drop to host-python when the user explicitly asks, or when Docker is demonstrably unavailable.
