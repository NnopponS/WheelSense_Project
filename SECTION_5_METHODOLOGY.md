5. หลักการ วิธีการ และขั้นตอนการทำงานของผลงาน

สถาปัตยกรรมของระบบ WheelSense (Wheelchair Sensing Platform) เป็นระบบติดตามตำแหน่งและสถานะของผู้ใช้วีลแชร์ภายในอาคาร (Indoor Localization & Telemetry) ที่พัฒนาขึ้นบนพื้นฐานของเทคโนโลยี Internet of Things (IoT) โดยมุ่งเน้นการใช้งานจริงที่มีประสิทธิภาพและต้นทุนต่ำ หลักการทำงานสำคัญคือการประยุกต์ใช้เทคนิค RSSI (Received Signal Strength Indicator) จากสัญญาณ Bluetooth Low Energy (BLE) ในการระบุตำแหน่ง ร่วมกับการประมวลผลเซนเซอร์ตรวจจับการเคลื่อนไหว (IMU) เพื่อวิเคราะห์พฤติกรรมของผู้ใช้งาน โดยข้อมูลทั้งหมดจะถูกส่งผ่านเครือข่าย WiFi ด้วยโปรโตคอล MQTT ไปยังเซิร์ฟเวอร์ส่วนกลางเพื่อจัดเก็บและแสดงผลแบบเรียลไทม์

ภาพที่ 4 แสดงภาพรวมสถาปัตยกรรมระบบของ WheelSense (System Architecture)

การพัฒนานวัตกรรม WheelSense ประกอบด้วย 2 ส่วนหลัก ได้แก่ ส่วนที่ 1 การพัฒนาฮาร์ดแวร์ (Hardware Development) และส่วนที่ 2 การพัฒนาซอฟต์แวร์ (Software Development) ดังนี้

ส่วนที่ 1 การพัฒนาฮาร์ดแวร์ (Hardware Development)

1) M5StickC (Wheelchair Gateway)
ทำหน้าที่เป็นอุปกรณ์ประมวลผลหลักที่ติดตั้งอยู่บนวีลแชร์ (Smart Gateway) โดยใช้ไมโครคอนโทรลเลอร์ ESP32 ที่มีประสิทธิภาพสูง ภายในมีการทำงานแบบ Multitasking ด้วยระบบปฏิบัติการ FreeRTOS เพื่อจัดการกระบวนการทำงานที่ซับซ้อน ได้แก่:
- BLE Scanner: ทำหน้าที่สแกนค้นหาสัญญาณจากอุปกรณ์ Node ที่ติดตั้งตามห้องต่าง ๆ ตลอดเวลา โดยตั้งค่า Scan Window ให้เหมาะสมเพื่ออ่านค่า RSSI สำหรับนำไปคำนวณหาระยะห่างและระบุตำแหน่งห้องที่ใกล้ที่สุด
- IMU Processing: อ่านค่าจากเซนเซอร์ความเร่ง (Accelerometer) และไจโรสโคป (Gyroscope) แบบ 6 แกน เพื่อคำนวณระยะทาง (Distance) และความเร็ว (Speed) ในการเคลื่อนที่ รวมถึงตรวจจับลักษณะการเคลื่อนไหว (Motion Detection) เช่น การหยุดนิ่ง การเคลื่อนที่ไปข้างหน้า หรือการหมุนตัว
- MQTT Client: เชื่อมต่อกับเครือข่าย WiFi เพื่อส่งข้อมูล Telemetry (ID, Timestamp, Location, Motion Status) ไปยัง MQTT Broker ทุก ๆ 1 วินาที เพื่อให้ข้อมูลมีความต่อเนื่องและเป็นปัจจุบัน

ภาพที่ 5 แสดงอุปกรณ์ M5StickC ที่ติดตั้งบนวีลแชร์

2) ESP32-S3 (Stationary Node)
ทำหน้าที่เป็นอุปกรณ์ระบุตำแหน่งอ้างอิง (Reference Nodes) ที่ติดตั้งประจำจุดต่าง ๆ ภายในอาคาร เช่น ห้องนอน ห้องน้ำ หรือทางเดิน โดยอุปกรณ์นี้จะทำงานในโหมด "BLE Advertiser" คือการปล่อยสัญญาณโฆษณา (Advertising Packet) ระบุชื่อ "WheelSense_ID" ออกมาอย่างต่อเนื่องด้วยความถี่สูง (Interval 100-200ms) เพื่อให้ M5StickC สามารถตรวจจับได้แม่นยำ อุปกรณ์นี้ถูกออกแบบให้ใช้พลังงานต่ำมาก (Low Power Consumption) สามารถติดตั้งได้ง่ายโดยไม่ต้องเดินสายสัญญาณเครือข่าย

ภาพที่ 6 แสดงอุปกรณ์ ESP32-S3 Node

ส่วนที่ 2 การพัฒนาซอฟต์แวร์ (Software Development)

1) Firmware Development
- M5StickC Firmware: พัฒนาด้วยภาษา C++ บน PlatformIO Framework โดยใช้ไลบรารี NimBLE สำหรับจัดการ Bluetooth Stack ที่มีประสิทธิภาพสูง ลดการใช้หน่วยความจำ และ ArduinoJson สำหรับจัดการข้อมูล JSON ในการส่งผ่าน MQTT พร้อมระบบ WiFi Manager สำหรับตั้งค่าเครือข่ายผ่านหน้าเว็บ
- ESP32-S3 Firmware: พัฒนาโปรแกรมเน้นความเสถียรในการปล่อยสัญญาณ Bluetooth (Beacon Mode) และการจัดการพลังงานเพื่อให้สามารถทำงานได้ยาวนาน

2) Backend Server Infrastructure
ระบบเซิร์ฟเวอร์ถูกออกแบบเป็นสถาปัตยกรรม Microservices ที่ทำงานอยู่บน Docker Container เพื่อความยืดหยุ่นในการติดตั้งและขยายระบบ (Scalability) ประกอบด้วย:
- MQTT Broker (EMQX Public / Mosquitto): ทำหน้าที่เป็นตัวกลางในการรับส่งข้อความ (Message Broker) ระหว่างอุปกรณ์ฮาร์ดแวร์และเซิร์ฟเวอร์ รองรับการเชื่อมต่อพร้อมกันจำนวนมากแบบ Asynchronous
- Data Collector Service: พัฒนาด้วยภาษา Python ใช้ไลบรารี Paho-MQTT เพื่อดักฟัง (Subscribe) ข้อมูลจากหัวข้อ "WheelSense/data" เมื่อได้รับข้อมูลจะทำการตรวจสอบความถูกต้อง (Data Validation) และจัดรูปแบบก่อนส่งต่อให้ API
- FastAPI Application: เป็นแกนกลางของระบบ (Core API) ที่พัฒนาด้วย Python FastAPI ทำหน้าที่ให้บริการ RESTful API สำหรับการจัดการข้อมูลทั้งหมด รวมถึงคำนวณตรรกะทางธุรกิจ (Business Logic) และเชื่อมต่อกับฐานข้อมูล
- SQLite Database: ระบบฐานข้อมูลแบบ Relational ที่มีความรวดเร็วและเบา เหมาะสำหรับการจัดเก็บข้อมูล Telemetry, ข้อมูลผู้ใช้งาน, และโครงสร้างแผนที่อาคาร (Map Data)

3) Frontend Dashboard (Web Application)
พัฒนาด้วย React Framework ร่วมกับ Vite Build Tool เพื่อให้ได้เว็บแอปพลิเคชันที่มีประสิทธิภาพสูง (High Performance) และรองรับการแสดงผลบนทุกอุปกรณ์ (Responsive Design) โดยมีฟีเจอร์สำคัญคือ:
- Real-time Dashboard: แสดงสถานะและตำแหน่งปัจจุบันของวีลแชร์บนแผนที่กราฟิก โดยใช้เทคนิค Polling หรือ WebSocket เพื่อดึงข้อมูลล่าสุดจาก API มาแสดงผลทันที
- Interactive Map Editor: เครื่องมือสร้างแผนที่อาคารแบบ Drag-and-drop ที่ช่วยให้ผู้ดูแลระบบสามารถวาดผังห้อง วางตำแหน่ง Node และกำหนดเส้นทางเดินได้เองผ่านหน้าเว็บ
- Analytics & History: ระบบกราฟรายงานผล (Charts) ที่แสดงสถิติการใช้งานย้อนหลัง เช่น ระยะทางรวม ความเร็วเฉลี่ย และประวัติการเข้าใช้งานแต่ละห้อง เพื่อนำไปวิเคราะห์สุขภาพและพฤติกรรมของผู้ป่วย

ภาพที่ 9 แสดงสถาปัตยกรรมซอฟต์แวร์ (Software Architecture)
ภาพที่ 10 แสดงหน้าจอเว็บไซต์ WheelSense Dashboard

ขั้นตอนการทำงานของระบบ (System Workflow)
1. การระบุตำแหน่ง (Localization): อุปกรณ์ Node (ESP32-S3) ปล่อยสัญญาณ BLE ตลอดเวลา เมื่อ M5StickC บนวีลแชร์เคลื่อนที่เข้าใกล้ จะรับสัญญาณและวัดค่าความแรง (RSSI) เพื่อระบุว่าอยู่ใกล้ Node ใดที่สุด
2. การประมวลผลเซนเซอร์ (Sensor Fusion): M5StickC อ่านค่า IMU เพื่อคำนวณการเคลื่อนไหวและทิศทาง ผสานกับข้อมูลตำแหน่งที่ได้จาก BLE
3. การส่งข้อมูล (Transmission): ข้อมูลทั้งหมดถูกแพ็คเป็น JSON และส่งผ่าน WiFi ไปยัง MQTT Broker
4. การจัดเก็บและวิเคราะห์ (Storage & Logic): Server รับข้อมูล นำไปบันทึกลงฐานข้อมูล และวิเคราะห์สถานะ (เช่น แจ้งเตือนหากพบนั่งนิ่งนานเกินไป)
5. การแสดงผล (Visualization): หน้าจอ Dashboard ดึงข้อมูลล่าสุดไปแสดงตำแหน่งบนแผนที่ เพื่อให้ผู้ดูแลรับทราบสถานะปัจจุบันของผู้ใช้งานได้ทันที
