# i18n Audit Plan — 100% EN/TH Parity (2026-04-20)

**Status: COMPLETE (Turn 8)**

Goal: every user-facing string in `frontend/app/**` and `frontend/components/**` is routed through `t(...)` with both `en` and `th` values in `frontend/lib/i18n.tsx`.

## Scope

Include:
- Visible text nodes in JSX.
- `placeholder`, `title`, `aria-label`, `alt`, `label` props on DOM/primitive elements.
- Toast and dialog strings (`sonner` calls).
- Form validation error messages originating on the client.

Exclude:
- Strings from the API (patient names, message bodies, audit payloads).
- Technical identifiers (tool names, device ids).
- Enum codes from DB (use explicit mapping to `t` keys, per `frontend/README.md`).

## Detection Method

Two complementary passes:

### Pass A — ripgrep heuristics
```
rg -n "placeholder=\"[^{]" frontend/app frontend/components
rg -n "aria-label=\"[^{]" frontend/app frontend/components
rg -n "title=\"[^{]" frontend/app frontend/components
rg -n ">[^<{}]*[A-Za-zก-๙][^<{}]*<" frontend/app frontend/components
```
Each hit that is not already `{t("...")}` is a candidate. Maintain results in `docs/plans/2026-04-20-i18n-hits.csv` per run.

### Pass B — runtime capture
Temporary dev-only guard in `lib/i18n.tsx` that logs any `t(key)` where `translations[key][locale]` is missing. Run `npm run build && npm run start`, click through every role, collect console warnings.

## Remediation Pattern

For each hit:
1. Choose key namespace per `frontend/README.md` conventions.
2. Add `{ en: "...", th: "..." }` to `translations` in `lib/i18n.tsx`.
3. Replace literal with `t("namespace.key")`.
4. Re-run Pass A until zero hits in the touched file.

## Completion Gates

- `rg` passes return zero hits for the scoped files (allow accepted list in `docs/plans/2026-04-20-i18n-allowlist.txt`).
- `npm run build` clean.
- Manual locale flip test: switch `ws_locale` to `th`, navigate every role root, no English residuals on primary surfaces.

## Work Batching

Split audit into 6 batches to keep PRs small:
1. Shared chrome (`RoleShell`, `RoleSidebar`, `TopBar`, `NotificationDrawer`).
2. Patient hub + sub-routes.
3. Observer dashboard + redesigned components.
4. Supervisor dashboard + tasks.
5. Head-nurse dashboard + personnel + tasks.
6. Admin residuals (less critical; do last).

Each batch: one commit, one build verification.
